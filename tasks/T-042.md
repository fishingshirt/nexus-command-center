# T-042 — Unified Notifications Center

## Goal
A central notification system that collects alerts from all apps (calendar reminders, todo due dates, agent messages, system alerts) into one slide-out panel.

## Acceptance Criteria
- [ ] Notification bell icon in header with badge count
- [ ] Slide-out panel from right: grouped by date, dismissible per-item or "clear all"
- [ ] Notification types: calendar event提醒, todo due soon, agent build complete, system low disk, etc.
- [ ] Browser push notification option: toggle in settings, requests permissions
- [ ] Persistent storage: `localStorage` key `ncc-notifications` (max 100, FIFO eviction)
- [ ] Mark read / unread: blue dot on unread, tap to mark read
- [ ] Agent integration: agent can post notifications to the panel directly
- [ ] Sound: optional soft chime on new notification (Web Audio, not external file)

## UI Details
- Bell icon: `bell` from Phosphor/Heroicons, red badge with white number
- Panel: header "Notifications" + gear icon → settings, list of cards
- Each card: app icon, message, relative time ("2 min ago"), dismiss ❌
- Empty state: "All caught up 🎉"

## Tech Notes
- API: `notify(title, body, app, priority='normal')` adds to panel + optional browser push
- `Notification.requestPermission()` on first toggle enable
- Sound: short synthesized beep (oscillator, 0.2s) to avoid file dependencies
- Auto-dismiss calendar reminders after event passes

## Agent Note
Build the notification API first (`public/js/notifications.js`), then wire calendar + todo to emit events. The agent stats panel (T-022) already has a notification pattern — reuse or replace with this unified system.
