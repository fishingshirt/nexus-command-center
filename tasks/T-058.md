# T-058: All Themes Should Support Dark Mode (Default On)

## Status
`PENDING`

## Goal
Currently only some themes are dark by default (`jarvis`, `midnight-hacker`, `retro-arcade`). The light themes (`professional`, `ocean-breeze`, `sakura-garden`) do not have dark mode variants. Add dark mode support to ALL themes and ensure dark mode is the default across the board.

## Acceptance Criteria
- [ ] Add dark mode CSS variables or `@media (prefers-color-scheme: dark)` overrides for:
  - `professional.css` (currently light #ffffff)
  - `ocean-breeze.css` (currently light #ecfeff)
  - `sakura-garden.css` (currently light #fff0f3)
- [ ] Each theme's dark mode should preserve the theme's unique accent color while using dark backgrounds
- [ ] Add a `data-theme-mode="dark"` or similar attribute system so JS can toggle between light/dark variants of the SAME theme
- [ ] Settings panel gets a toggle for "Dark mode" that applies globally regardless of which theme is selected
- [ ] Default state: dark mode is ON for all themes
- [ ] When switching themes, if dark mode is enabled, the dark variant of the new theme activates automatically
- [ ] Store `themeMode: 'dark'` in localStorage by default (alongside `theme`)

## UI Details
- Dark mode toggle in Settings → Appearance, placed right below the theme selector
- Toggle label: "Dark mode" with a moon icon
- When toggled, CSS variables switch instantly via a class on `<html>` or `<body>`

## Tech Notes
- Option A: Extend each theme file with `body[data-theme-mode="dark"]` overrides
- Option B: Create companion `-dark.css` files for light themes and swap them dynamically
- Option A is preferred (single file per theme, less HTTP requests)
- Ensure smooth transition: `transition: background-color 0.3s ease, color 0.3s ease`

## Files Expected to Touch
- `public/css/themes/professional.css`
- `public/css/themes/ocean-breeze.css`
- `public/css/themes/sakura-garden.css`
- `public/css/base.css` (global dark-mode transition utilities)
- `public/js/settings.js` (dark mode toggle, storage)
- `public/js/app.js` (apply dark mode on load)
- `public/index.html` (default data-theme-mode attribute)

## Dependencies
- T-057: Default theme should be dark (do T-057 first to set the dark default infrastructure)
