# T-032-a: Gmail OAuth2 Backend

## Status
IN_PROGRESS

## Goal
Add Gmail OAuth2 endpoints to `nexus-server.py` mirroring the calendar OAuth pattern.

## Acceptance Criteria
- [ ] `GET /api/email/oauth/start` returns OAuth URL with `https://www.googleapis.com/auth/gmail.modify` scope
- [ ] `GET /api/email/oauth/callback` exchanges code for tokens and saves to `~/.hermes/nexus-gmail-tokens.json`
- [ ] `GET /api/email/status` returns `{ linked: bool, email: string|null }`
- [ ] `POST /api/email/unlink` deletes token file
- [ ] Token refresh logic reuses calendar `_refresh_token` helper or mirrors it
- [ ] No secrets committed; reads from `~/.hermes/.env`

## Files to Touch
- `scripts/nexus-server.py` — add `/api/email/*` routes, token helpers

## Notes
- Reuses `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` from env
- Token file: `~/.hermes/nexus-gmail-tokens.json`
- Refresh helper: `_gmail_access_token()` similar to `_gcal_access_token()`
