# T-055-c: Feedback Pipeline — Agent Integration + Settings

## Goal
Wire feedback queue into agent wake cycle and settings.

## Acceptance Criteria
1. In agent wake-cycle script (or Python helper read by agent):
   - After normal task work, read `data/feedback-queue.jsonl` for `status:pending` entries.
   - Classify each: feature/bug/improvement/theme/other.
   - Heuristic: if description ≥40 chars and priority must/should → auto-generate task.
   - Bug → always auto-generate B-XXX.
   - Theme → auto-generate if mentions colors/vibe.
   - Vague → status=review.
   - Auto-generation: write `tasks/T-XXX.md`, update WHITEBOARD.md, commit+push, update entry to `tasked` with `linkedTask`.
   - Send Telegram notification to user.
   - If auto-generate OFF, always status=review.
2. Settings toggles in `public/js/settings.js`:
   - "Auto-generate tasks from feedback" (default ON)
   - "Notify me when feedback is processed" (default ON)
3. Notification toast when feedback status changes (wire into `notifications.js`).

## Files Expected to Touch
- Agent Python script / nexus-server.py (agent command endpoint)
- `public/js/settings.js`
- `public/js/notifications.js`

## Dependencies
T-055-a, T-055-b, T-042 Notifications
