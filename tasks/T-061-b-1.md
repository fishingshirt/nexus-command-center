## Data Schema

```js
{
  id: string (crypto.randomUUID || Date.now() + Math.random().toString(36).slice(2)),
  title: string,
  url: string,
  image: string,
  price: number | null,
  currency: 'USD' | 'EUR' | 'GBP' | 'CAD' | 'JPY' | 'Other',
  priority: 'Low' | 'Normal' | 'High' | 'Must Buy',
  status: 'Want' | 'Watching' | 'Purchased' | 'Archived',
  tags: string[],
  notes: string,
  created: ISO string,
  updated: ISO string
}
```

- `tags` stored as comma-separated input, parsed on save.
- `image` falls back to generic placeholder icon if missing.

## UI Details

### Add Modal
- Triggered by `#wishlist-add-btn` click.
- Form fields: Title (required), URL, Image URL, Price, Currency select, Priority select, Tags (comma-separated), Notes (textarea).
- Actions: Save (primary), Cancel (secondary).
- On save: validate title, generate ID, push to items array, save to localStorage, close modal, re-render grid, toast "Added to wishlist".

### Card Grid
- CSS grid: 1 col mobile, 2 tablet, 3 desktop.
- Each card: top image (16:9 aspect ratio, `object-fit: cover`, lazy load placeholder), priority color strip (4px left border), title, price badge (e.g. "$120.00"), status pill (`Want` default), actions row (Edit placeholder, Delete button).
- Delete: immediate removal + toast "Item deleted". No undo for this sub-task (T-061-b-3 may add undo).

### Empty State
- Already exists in HTML. Hide grid when items length === 0.
