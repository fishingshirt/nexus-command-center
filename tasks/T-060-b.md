# T-060-b — QA Agent Review Checklist & Sign-Off System

## Status
`PENDING`

## Goal
Build the Stage 2 quality approval system. When QC passes, the agent reads a generated checklist and must verify everything before signing off.

## Acceptance Criteria
- [ ] `scripts/nexus-qa-checklist.py` created
- [ ] Reads `~/.hermes/nexus-quality-queue.json`, finds entries with `QC-PASSED`
- [ ] Generates a markdown checklist file `~/.hermes/qa-checklist-XXX.md` with per-file review items
- [ ] Checklist includes:
  - Logic & edge case review (state keys, null refs, async handling)
  - Functional test plan (feature area, 3 edge cases, mobile + desktop, 2 themes, offline)
  - Site load verification (no console errors, no FOUC, SW registers, all apps visible)
  - Regression check (run `validate-js.js`, run curl smoke tests, server starts clean)
- [ ] After agent completes checklist, they write `QA-XXX.json` to the queue with `overall: PASSED|FAILED`
- [ ] If FAILED, entry reverts to `QC-FAILED` and gets re-queued for fix
- [ ] If PASSED, entry advances to `MERGING` state

## Output Format
```json
{
  "id": "QA-001",
  "timestamp": "2026-05-10T14:00:00Z",
  "parent_qc": "QC-001",
  "commit": "HEAD",
  "checks": [
    {"check": "logic-review", "status": "pass", "detail": "3 files reviewed, no race conditions"},
    {"check": "functional-test", "status": "pass", "detail": "Mobile OK, desktop OK, offline OK"}
  ],
  "overall": "PASSED",
  "signed_by": "agent",
  "signed_at": "2026-05-10T14:30:00Z"
}
```

## Agent Note
This script does NOT do the testing — it generates the checklist that the agent follows. The actual sign-off is manual (agent-written). This is intentional because automated QA is not good enough for logic validation.
