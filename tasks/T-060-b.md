# T-060-b — QA Agent Review Checklist & Sign-Off System

## Status
`DONE`

## Goal
Build the Stage 2 quality approval system. When QC passes, the agent reads a generated checklist and must verify the **nuanced, human-level aspects** that automated QC cannot catch — logic edge cases, UX flow, visual regressions, and real-world functional behavior.

## Philosophy
- **QC stops mechanical bugs. QA stops logical bugs.**
- If a bug reaches the live dashboard, both stages are at fault — tighten the checklists.
- QA is a mandatory human-in-the-loop gate before merge. No exceptions.

## Acceptance Criteria
- [x] `scripts/nexus-qa-checklist.py` created
- [x] Reads `~/.hermes/nexus-quality-queue.json`, finds entries with `QC-PASSED`
- [x] Generates a markdown checklist file `~/.hermes/qa-checklist-XXX.md` with per-file review items
- [x] Checklist includes:
  - [x] Logic & edge case review (state keys, null refs, async handling)
  - [x] Functional test plan (feature area, 3 edge cases, mobile + desktop, 2 themes, offline)
  - [x] Site load verification (no console errors, no FOUC, SW registers, all apps visible)
  - [x] Regression check (run `validate-js.js`, run curl smoke tests, server starts clean)
- [x] After agent completes checklist, they write `QA-XXX.json` to the queue with `overall: PASSED|FAILED`
- [x] If FAILED, **bounce back to task**: the change is NOT kept in the pipeline. The agent must:
  1. Read the QA failure report.
  2. Go back to the original task file (`tasks/T-XXX.md`).
  3. Fix the bugs.
  4. Re-enter QC from the beginning.
- [x] If PASSED, entry advances to `MERGING` state

## Output Format
```json
{
  "id": "QA-001",
  "timestamp": "2026-05-10T14:00:00Z",
  "parent_qc": "QC-001",
  "commit": "HEAD",
  "checks": [
    {"check": "logic-review", "status": "pass", "detail": "3 files reviewed, no race conditions"},
    {"check": "functional-test", "status": "pass", "detail": "Mobile OK, desktop OK, offline OK"}
  ],
  "overall": "PASSED",
  "signed_by": "agent",
  "signed_at": "2026-05-10T14:30:00Z"
}
```

## Agent Note
This script does NOT do the testing — it generates the checklist that the agent follows. The actual sign-off is manual (agent-written). This is intentional because automated QA is not good enough for logic validation.
