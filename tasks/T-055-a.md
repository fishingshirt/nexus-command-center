# T-055-a: Feedback Pipeline — Backend Queue API

## Goal
Build server-side feedback queue endpoints.

## Acceptance Criteria
1. In `scripts/nexus-server.py`, add:
   - `POST /api/feedback` — body `{type, title, description, priority, answers[], userId?, timestamp}`. Append to `data/feedback-queue.jsonl`. Return `{ok:true, id}`.
   - `GET /api/feedback/:id` — return single entry.
   - `GET /api/feedback/list?status=...` — return `{items:[], total}`.
   - `PATCH /api/feedback/:id/status` — body `{status, linkedTask?, note?}` (agent-only).
2. Validation: reject empty title or description (max title 200 chars).
3. On startup, create `data/feedback-queue.jsonl` if missing.
4. Add `data/feedback-queue.jsonl` to `.gitignore`.

## Files Expected to Touch
- `scripts/nexus-server.py`
- `.gitignore`

## Dependencies
None
