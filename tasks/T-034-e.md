# T-034-e — Scope Upgrade & Re-authorize Prompt

## Goal
When the user links Google Calendar with a read-only scope (or the backend detects `readOnly: true`), show a persistent **Re-authorize** button in Settings so they can re-trigger OAuth with the full `https://www.googleapis.com/auth/calendar` scope.

## Acceptance Criteria
- [x] Settings → Google Calendar Sync section shows a **"Re-authorize for full access"** button when `/api/calendar/status` returns `readOnly: true`.
- [x] Clicking it opens the same OAuth popup as the Connect button.
- [x] After successful re-auth, the status badge updates to `synced` automatically.
- [x] The re-auth button is hidden when scope is already full write.

## Files to Touch
- `public/index.html` — add `#btn-sync-reauth` button next to Unlink.
- `public/js/app.js` — wire `btn-sync-reauth` click handler, show/hide logic, `checkStatus()` update.
- `public/css/base.css` — add `.btn-warning` style (amber) for the re-auth button.
