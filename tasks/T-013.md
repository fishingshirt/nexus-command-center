# T-013 — Data Persistence Layer

## Goal
Migrate all Nexus app data from `localStorage` to a real server-side JSON store (SQLite-ready), while keeping a transparent offline fallback so the dashboard works without the server.

---

## Sub-Tasks

### T-013-a — StorageAdapter abstraction + server endpoints
- **Files:**
  - `public/js/lib/storage-adapter.js` (new)
  - `scripts/nexus-server.py` (add `do_POST` / `do_GET` handlers)
- **API Endpoints:**
  - `GET /api/store/read?app=<id>` → returns `{"ok":true, "data":{...}}` or `{"ok":false, "error":"..."}`
  - `POST /api/store/write` with JSON body `{app, data}` → returns `{"ok":true}`
  - `POST /api/store/merge` with JSON body `{app, patch}` → server-side deep-merge, returns merged data
- **StorageAdapter JS contract:**
  ```javascript
  export const storage = {
    async read(app) { /* try server → fallback localStorage */ },
    async write(app, data) { /* try server → fallback localStorage */ },
    async merge(app, patch) { /* try server merge → local merge */ },
    isOnline() { return navigator.onLine; }
  };
  ```
- **localStorage keys:** prefix remains `ncc-`. Server file path: `~/.hermes/nexus-store/<app>.json`
- **Fallback behavior:** If server returns non-200 or timeout >2s, read/write to `localStorage` under same `ncc-<app>` key. Set `ncc-store-mode` to `"offline"` so UI can show an offline badge.
- **Server-side:** Use Python `json` module, write atomically (temp file + rename). One JSON file per app. Directory auto-created on first write.

### T-013-b — Per-app migration script (one-time data lift)
- **File:** `public/js/lib/migrate-legacy.js` (new)
- **Behavior:** On first StorageAdapter init, read ALL known `localStorage` keys (`ncc-calendar`, `ncc-notes`, `ncc-todo`, `ncc-finance`, `ncc-weather`, `ncc-phone`, `ncc-backup`, `ncc-auth`, `ncc-settings`, `ncc-worldclock`, `ncc-pomodoro`, `ncc-gcal-sync`, `ncc-arcade`) and PUSH them to server via `/api/store/write`.
- **Flag:** Set `localStorage.setItem('ncc-migrated-v1', 'true')` so migration never runs twice.
- **UI:** Show a single toast `"Migrating data to server…"` then `"Migration complete"`.

### T-013-c — Audit all existing apps for raw localStorage usage
- **Files to inspect:**
  - `public/js/apps/calendar.js`
  - `public/js/apps/notes.js`
  - `public/js/apps/todo.js`
  - `public/js/apps/weather.js`
  - `public/js/apps/phone-bridge.js`
  - `public/js/apps/backup.js`
  - `public/js/apps/auth.js`
  - `public/js/apps/finance.js`
  - `public/js/apps/worldclock.js`
  - `public/js/apps/pomodoro.js`
  - `public/js/apps/gcal-sync.js`
  - `public/js/apps/arcade.js`
- **Replace pattern:** Every `localStorage.getItem('ncc-...')` and `localStorage.setItem('ncc-...', ...)` inside data-producing apps should route through `storageAdapter.read(app)` / `storageAdapter.write(app, data)`.
- **Settings exception:** `public/js/app.js` may keep direct `localStorage` for `ncc-settings` and `ncc-theme` as boot-time preferences that must resolve synchronously (SettingsAdapter can still use adapter but with sync-fallback).
- **Batch-replace strategy:** Use the batch JS patcher pattern (one `execute_code` script) to do exact string replacements across all `apps/*.js` files in one pass.

### T-013-d — Settings sync bridge
- **Files:**
  - `public/js/app.js`
- **Behavior:** Settings are read synchronously at boot to avoid FOUC. Keep `localStorage` as a fast synchronous cache layer. Add a background `storageAdapter.write('settings', settings)` after every `saveSettings()` call so the server copy stays in sync. On server read, merge server copy over local cache (server wins on conflict).

### T-013-e — Offline mode badge + auto-retry queue
- **Files:**
  - `public/js/app.js` (add offline indicator wiring)
  - `public/css/base.css` (offline badge styles)
- **Behavior:**
  - When `storage.write()` falls back to localStorage, set `window.__nexusOffline = true`.
  - The existing `#offline-indicator` span in the header should show `"Offline — syncing when back"`.
  - On `window 'online'` event, attempt to flush all `ncc-*` localStorage keys to server. Clear indicator on success.
- **Queue:** Simple in-memory array `window.__offlineQueue = [{app, data}, ...]`. Flushed in order, one at a time.

### T-013-f — Data integrity guard (JSON parse safety)
- **Files:**
  - `public/js/lib/storage-adapter.js`
- **Behavior:**
  - Wrap every `JSON.parse` in `try/catch`. On parse error, return `{}` and log `[StorageAdapter] Corrupted data for ${app}`.
  - On server write, validate data is plain JSON-serializable (no functions, no circular refs). If invalid, reject with error toast.
  - If server read returns `{}` but localStorage has data, prefer localStorage and backfill server silently.

## Acceptance Criteria (T-013 umbrella)
- [ ] All apps use `StorageAdapter` for data CRUD (settings除外 synchronous cache)
- [ ] Server-side JSON files exist under `~/.hermes/nexus-store/`
- [ ] Dashboard works fully offline with no server — data reads from localStorage
- [ ] Data survives browser cache clear because server has the master copy
- [ ] One-time migration runs on first boot after deploy and then never again
- [ ] Offline indicator visible when server is unreachable
- [ ] No `console.error` in new code; `console.warn` only for recoverable failures
