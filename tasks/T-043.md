# T-043 — Quick Capture Widget (Voice + Fast Entry)

## Goal
A persistent floating action button (FAB) or swipe-up panel for rapid data entry: quick note, calendar event, todo task, or voice memo.

## Acceptance Criteria
- [ ] FAB: bottom-right circular + button, expands into radial menu (note, event, todo, voice)
- [ ] Quick note: text area → saves to Notes app instantly
- [ ] Quick event: title + date + time → saves to Calendar + Google Calendar if linked
- [ ] Quick todo: text → saves to default "Today" list
- [ ] Voice memo: hold mic button → record via Web Audio API → save as blob in `localStorage` (or transcript if speech-to-text available)
- [ ] Keyboard shortcut: `N` opens note, `T` opens todo, `C` opens event
- [ ] Mobile: swipe up from bottom edge to reveal capture bar
- [ ] Agent command: "quick note: buy milk tomorrow" → agent posts to Quick Capture queue

## UI Details
- FAB closed: solid accent color, white + icon
- FAB open: 4 satellite buttons around center with labels + icons
- Capture modal: slides up from bottom, dark backdrop, minimal fields
- Voice state: red recording pulse, timer, tap to stop → waveform preview

## Tech Notes
- `MediaRecorder` API for voice (check `MediaRecorder.isTypeSupported('audio/webm')`)
- Store voice memos in `localStorage`? They're large — store metadata there, dump blob to `IndexedDB` if available.
- Speech-to-text: use `webkitSpeechRecognition` if available (Chrome), else record only.

## Agent Note
This is a high-value daily-use feature. Keep it fast — under 3 taps to capture anything. Reuse the Notes/Calendar/Todo storage schemas; don't create new ones.
