# T-041 — Global Search Bar

## Goal
A single search bar (pinned to top of dashboard or triggered via `/` hotkey) that searches across notes, todos, calendar events, and bookmarks.

## Acceptance Criteria
- [ ] Search input: fixed top bar or floating modal triggered by `/` or tap on 🔍 icon
- [ ] Indexed search: build a lightweight in-memory index on boot from all app data stores
- [ ] Results grouped by app: "Calendar — team standup tomorrow", "Note — grocery list", "Todo — fix auth bug"
- [ ] Keyboard navigation: arrow keys to navigate results, Enter to open, Escape to close
- [ ] Fuzzy matching: tolerate typos (simple Levenshtein or `includes()` fallback)
- [ ] Mobile: full-screen overlay with back button, soft keyboard auto-focus
- [ ] Agent command: "search for standup" returns results in Telegram chat

## UI Details
- Input placeholder: "Search everything..."
- Result row: app icon + title + matched snippet + timestamp
- Empty state: "No results for 'xyz'"
- Max 10 results per category, "Show more" if needed

## Tech Notes
- Index builder runs in a `requestIdleCallback` after `initApp()` so it doesn't block boot.
- Index format: `{notes: [{id, title, body, updatedAt}], todos: [...], calendar: [...]}`
- Debounce input at 150ms
- No external search library — pure JS.
