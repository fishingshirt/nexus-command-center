# T-031-a: Cloud Vault — JS App Module + Wiring

## Goal
Build the Cloud Vault frontend app: file listing, upload (drag-and-drop), folder tree, search, download trigger.

## Acceptance Criteria
1. Create `public/js/apps/vault.js` with:
   - App lifecycle: `initApp`, `showApp`, `hideApp`
   - State in `localStorage` key `ncc-vault-meta` (cache of file list from server)
   - Upload: drag-and-drop zone + file input. On drop/read, POST file to `/api/vault/upload`
   - Folder tree sidebar: expandable folders, click to filter list
   - File list grid/table: filename, size, type icon, folder badge, uploaded date
   - Search: real-time filter by filename
   - Download button per file: GET `/api/vault/download?id=`
   - Context menu (or button row): move to folder, delete
   - Empty state: "Drop files here or tap Upload"
2. Wire in `public/index.html`:
   - Add `<div id="view-vault" class="app-view" aria-label="Cloud Vault">...</div>` inside `<main>`
   - Add nav link `<li><a href="#vault" data-app="vault">🔐 Vault</a></li>` inside `#app-nav ul`
   - Add `<script type="module" src="js/apps/vault.js"></script>` before closing `body`
3. Wire in `public/js/app.js`:
   - Import `initVault` and call it in `initApp()` (after other app inits)
   - Add `vault` entry to any app registry / nav handler
4. File icons: use emoji or SVG based on mime type (image/pdf/code/audio/zip/other)

## Files Expected to Touch
- `public/js/apps/vault.js` (new)
- `public/index.html` (view block + nav link + script tag)
- `public/js/app.js` (registry + init wiring)

## Dependencies
None
