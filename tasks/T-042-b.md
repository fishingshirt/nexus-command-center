# T-042-b: Notification Bell UI + Slide-out Panel

## Status
`PENDING`

## Goal
Build the visual notification center: bell in header, badge count, slide-out drawer panel.

## Acceptance Criteria
- [ ] Bell icon (SVG) in `.header-right` next to search button
- [ ] Red badge with white number (`#notification-badge`) — hidden when 0
- [ ] Click bell → slide-out panel from right (`#notification-panel`) with `transform: translateX(0)` animation
- [ ] Panel header: "Notifications" + gear icon → links to Settings
- [ ] Panel body: grouped by relative date header ("Today", "Yesterday", "Earlier")
- [ ] Each card: app icon emoji, message text, relative time ("2 min ago"), dismiss ❌ button
- [ ] Empty state: "All caught up 🎉" centered in panel
- [ ] Unread items have blue left border/dot (`.notification-unread`)
- [ ] Tap card → mark read
- [ ] "Clear all" button at bottom of panel
- [ ] Focus trap: first focusable element on open, Escape to close

## Files Expected to Touch
- `public/index.html` — add bell button + panel markup
- `public/css/base.css` — panel styles (keep within ~150 lines)
- `public/js/app.js` — wire bell click, panel open/close, badge update
- `public/js/notifications.js` — consume API from T-042-a

## CSS Custom Properties
- Use `--panel-width: 360px` (max 90vw on mobile)
- Backdrop: `--backdrop: rgba(0,0,0,0.4)`

## Responsive
- 375px: panel full-width or 90vw, touch-friendly tap targets (min 44px)
- 768px+: panel 360px wide

## Accessibility
- `aria-label="Notifications"` on bell
- `role="dialog" aria-modal="true"` on panel
- `aria-live="polite"` on badge so screen readers announce count changes

## Test
- Open panel → bell badge resets to 0
- Add notification via console → badge increments, panel shows card
- Dismiss single card → removed from panel + localStorage
- Clear all → empty state visible

## Dependencies
T-042-a (notification API core)
