# T-040-a-1: Widget Grid Core Engine + Default Layout

## Goal
Build the widget grid engine module.

## Acceptance Criteria
1. Create `public/js/widgets/grid.js` with:
   - `WidgetGrid(containerEl, registry)` class
   - Manages a CSS-grid container (class `widget-grid`)
   - Persists layout to `localStorage` key `ncc-dashboard-widgets`
   - `registerWidget(type, factoryFn)` — apps call this to expose widgets
   - `renderLayout()` — reads saved layout or uses default, creates card DOM
   - `enterEditMode()` / `exitEditMode()` — toggles drag handles and dim overlay
   - Drag-and-drop reorder via HTML5 Drag API
   - Default layout: `[{type:'weather'},{type:'calendar-today'},{type:'todo-count'},{type:'agent-stats'}]`
   - Auto-refresh every 60 s per widget via `setInterval`
   - Cleanup method `destroy()` clearing intervals and listeners
2. Create `public/css/widgets/grid.css` (new) with `.widget-grid` styles, `.widget-card`, drag placeholders, edit-mode overlay. All colours via CSS custom properties.
3. Wire CSS into `public/index.html` via `<link rel="stylesheet" href="css/widgets/grid.css">`.

## Files Expected to Touch
- `public/js/widgets/grid.js` (new)
- `public/css/widgets/grid.css` (new)
- `public/index.html`

## Dependencies
None.
