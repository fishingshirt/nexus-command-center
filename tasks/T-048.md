# T-048 — Usage Analytics & Screen Time

## Goal
Track and visualize how the user spends time in the dashboard — which apps are used most, daily streaks, peak usage hours.

## Acceptance Criteria
- [ ] Passive tracking: log timestamp + app opened + duration on every app switch
- [ ] Dashboard grid card: "Today's usage" pie chart (SVG) with top 3 apps
- [ ] Full analytics app:
  - Daily/weekly/monthly view toggle
  - Bar chart: hours per day
  - Stacked bar: breakdown by app
  - Peak hours heatmap
  - Longest streak (consecutive days with 1+ session)
- [ ] Privacy: all data local only, no external transmission, exportable JSON
- [ ] Agent access: agent can read analytics JSON to suggest productivity improvements
- [ ] Opt-out: toggle in settings to disable tracking entirely

## UI Details
- Widget: compact donut chart (SVG ring), center shows total minutes today
- Full app: clean data-viz layout, chart.js-like but hand-rolled SVG since no deps
- Heatmap: 7x24 grid of colored squares (days × hours), color intensity = usage
- Empty state: "Start using apps to see your stats 📊"

## Tech Notes
- Storage: `localStorage` `ncc-analytics-sessions` with `{date, app, minutes}` aggregates
- Daily aggregation runs on boot to keep data compact
- SVG charts: `<rect>` bars, `<path>` for donut segments
- No canvas — keeps consistent with theme system via CSS variables
- Data retention: auto-prune entries older than 90 days
