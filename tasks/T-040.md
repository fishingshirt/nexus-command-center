# T-040 — Customizable Dashboard Widget Grid

## Goal
Replace the static home screen with a drag-and-drop widget grid. Users can add, rearrange, resize, and remove widget cards that surface live data from other apps.

## Acceptance Criteria
- [ ] Widget registry: each app exposes one or more widgets (e.g. calendar "Today's Events", todo "Pending Count", weather "Current Conditions")
- [ ] Dashboard edit mode: long-press or edit button enters arrangement mode — drag to reorder, tap 🗑️ to remove, tap ➕ to add
- [ ] Add Widget drawer: scrollable list of available widgets with preview
- [ ] Layout persistence: widget positions saved to `localStorage` under `ncc-dashboard-widgets`
- [ ] Responsive grid: 1 column on mobile, 2 on tablet, 3-4 on desktop
- [ ] Default layout: pre-populated with weather, calendar today, todo count, agent stats on first boot
- [ ] Widget lifecycle: each widget auto-refreshes its data every 60 seconds without full page reload
- [ ] Agent command: "add calendar widget to my dashboard" or "remove weather widget"

## UI Details
- Widget card: rounded rectangle, app-colored top border, title bar with icon, refresh spinner on update
- Empty state: "Tap + to add your first widget" with animated pulse on the add button
- Edit mode overlay: subtle dim behind cards, drag handles appear, done button in top-right

## Tech Notes
- Use CSS grid with `grid-template-columns: repeat(auto-fill, minmax(280px, 1fr))`
- Drag-and-drop: native HTML5 Drag API (no library).
- Each widget is a factory function: `const widget = createWidgetCard(type, config) → HTMLElement`
- Store layout as JSON: `[{type: "weather", position: 0, config: {cityId: ...}}]`

## Agent Note
This touches `index.html`, `app.js`, `base.css`, and adds `public/js/widgets/`. Keep widgets thin — they query existing app data stores, not duplicate logic. Reuse existing render helpers from calendar.js, todo.js, etc.
