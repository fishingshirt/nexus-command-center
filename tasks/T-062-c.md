# T-062-c: Dashboard Category Sections — Recipe as Standalone Category

## Goal
Restructure the main dashboard grid into visual category sections so that the Recipe App is its own standalone category, and other apps are grouped logically. Do NOT change any app functionality or remove any cards.

## Acceptance Criteria
- [ ] Dashboard gets sectioned into category groups with visual headers:
  1. **Recipes** — standalone, only the Recipe card (can span full width or sit in its own row)
  2. **Productivity** — Calendar, Notes, To-Do, Email, RSS Reader
  3. **Lifestyle** — Weather, World Clock, Focus (Pomodoro), News Hub
  4. **Finance** — Finance, Budget
  5. **Tools** — Phone Bridge, PDF Editor, Vault, Wishlist, Feedback
  6. **Entertainment** — Arcade
- [ ] Each category header uses an `<h3 class="dash-cat-title">` inside a wrapper `<div class="dash-cat">`.
- [ ] Category wrappers use `grid-column: 1 / -1` so they span the full width of `.app-grid`.
- [ ] The Recipe category appears FIRST (at the top) — it is the hero category.
- [ ] All app cards keep their exact existing markup (icon, badge, aria-label, etc.).
- [ ] CSS additions in `public/css/base.css`:
  - `.dash-cat { grid-column: 1 / -1; margin-bottom: var(--space-2); }`
  - `.dash-cat-title { font-size: var(--font-size-sm); color: var(--text-muted); text-transform: uppercase; letter-spacing: 1px; margin: var(--space-2) 0 var(--space-1); }`
  - `.dash-cat-grid { display: grid; grid-template-columns: repeat(2, 1fr); gap: var(--space-4); }`
  - Responsive breakpoints identical to `.app-grid` (640px → 3 cols, 1024px → 4 cols).
- [ ] Sidebar nav gets a Recipe link in its own group (after Vault, before Settings), with a `nav-divider` before it.
- [ ] No app JavaScript logic changes.
- [ ] No existing `.app-grid` CSS rules are removed or altered.

## Files to Touch
- `public/index.html` — wrap app cards in category sections, add sidebar nav link
- `public/css/base.css` — add category section styles

## Pitfall Avoidance
- Do NOT delete or modify any existing app-card markup — only wrap it.
- The `.app-grid` CSS must remain unchanged; use nested grids inside `.dash-cat`.
- Keep the weather-pill-row and clock-pill-row inside `.app-grid` (they render inline, not as cards).
