# T-040-a-2: Widget Grid Dashboard Wiring + Widget Registration Stubs

## Goal
Wire the widget grid into the dashboard view and register initial widget factories.

## Acceptance Criteria
1. In `public/index.html`:
   - Add `<div id="widget-grid" class="widget-grid"></div>` inside `#view-dashboard`
   - Add `<script type="module" src="js/widgets/grid.js"></script>` before `app.js` script tag
2. In `public/js/app.js`:
   - Import `WidgetGrid` from `'./widgets/grid.js'`
   - After `initApp()`, instantiate `new WidgetGrid(document.getElementById('widget-grid'))`
   - Call `widgetGrid.renderLayout()` and start auto-refresh interval
3. In `public/js/apps/weather.js`, `calendar.js`, `todo.js`:
   - Add `import { widgetRegistry } from '../widgets/grid.js'` (or expose via window if import cycle risk)
   - Call `widgetRegistry.registerWidget(type, factoryFn)` with a minimal stub that returns a DOM fragment
   - Weather stub: shows city name + temp
   - Calendar stub: shows today's event count
   - To-Do stub: shows pending task count

## Files Expected to Touch
- `public/index.html`
- `public/js/app.js`
- `public/js/apps/weather.js`
- `public/js/apps/calendar.js`
- `public/js/apps/todo.js`

## Dependencies
T-040-a-1 (Widget Grid Core Engine) must be DONE first.
