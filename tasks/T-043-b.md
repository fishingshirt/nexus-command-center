# T-043-b — Quick Capture: Note + Event + Todo Logic

## Goal
Wire the capture modal buttons to save quick notes, calendar events, and todo tasks instantly into their respective apps.

## Files To Touch
1. `public/js/apps/quick-capture.js` — main capture logic
2. `public/js/apps/notes.js` — ensure it exports `addQuickNote(text)` or uses standardized `ncc-notes` schema
3. `public/js/apps/todo.js` — ensure it exports `addQuickTask(text)` or uses standardized `ncc-todo` schema
4. `public/js/apps/calendar.js` — ensure it exports `addQuickEvent(title, date, start, end)` or standardized schema
5. `public/js/app.js` — import and register `initQuickCapture` in the app registry

## Acceptance Criteria
- [ ] Clicking satellite "Note" opens modal with title + textarea; on save, appends to `ncc-notes` array with `{ id, title, body, createdAt, updatedAt, tags: [] }`
- [ ] Clicking satellite "Event" opens modal with title, date picker, start time, end time; on save, appends to `ncc-calendar-events` with standard event schema including `category: 'quick'`
- [ ] Clicking satellite "To-Do" opens modal with text; on save, appends to `ncc-todo` with `{ id, text, done: false, priority: 'normal', list: 'Today', createdAt, dueDate: null, tags: [] }`
- [ ] All saves trigger `toast('Saved to Notes' / 'Saved to Calendar' / 'Added to To-Do')`
- [ ] After save, modal closes automatically and the relevant app's view refreshes if it has a live refresh mechanism (best-effort: dispatch a custom event or rely on next load)
- [ ] `localStorage` key prefixes: `ncc-notes`, `ncc-todo`, `ncc-calendar-events` (reuse existing)
- [ ] Date input must use deterministic calendar logic: `const [y, m, d] = dateStr.split('-').map(Number); const dateObj = new Date(y, m - 1, d);`
- [ ] Event save must validate start < end; show toast error if not
- [ ] Note save must support empty title (auto-generate "Quick Note [timestamp]")

## Tech Notes
- Use `import { loadSettings, toast } from '../app.js';` in quick-capture.js
- Use existing note structure from `notes.js`: `getNotes()` returns parsed array; `saveNotes(notes)` persists
- Use existing todo structure from `todo.js`: `getTasks()` returns parsed array; `saveTasks(tasks)` persists
- Calendar event schema: `{ id, title, date, start, end, allDay: false, category: 'quick', description: '', recurrence: null, createdAt }`

## Edge Cases
- If notes/todo/calendar module functions are not exported, duplicate the minimal save logic locally (no external dependency)
- Gracefully handle `localStorage` quota full for large note bodies (try/catch around setItem)
