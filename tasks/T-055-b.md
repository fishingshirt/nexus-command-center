# T-055-b: Feedback Pipeline — Dashboard UI + Tracker Panel

## Goal
Overhaul the feedback form and add a submission tracker.

## Acceptance Criteria
1. In `public/js/app.js` (or `feedback.js`), refactor `initFeedback()`:
   - On submit, POST to `/api/feedback` instead of localStorage.
   - After submit, show "Submission Tracker" panel below form.
   - Tracker: list past submissions fetched from `GET /api/feedback/list`.
   - Each row: title, type icon, priority badge, status chip (pending/review/tasked/closed).
   - Tap row → expand detail view.
2. Status chips: `.chip-pending`, `.chip-review`, `.chip-tasked`, `.chip-closed`.
3. Badge on Feedback nav item shows count of `pending` + `review`.
4. Empty state: "No submissions yet. Shape the Nexus above. 💡"

## Files Expected to Touch
- `public/js/app.js` (or `public/js/apps/feedback.js`)
- `public/index.html` (tracker markup below feedback form)
- `public/css/apps/feedback.css` (new or update)
