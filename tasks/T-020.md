# T-020: AI Task Suggester

## Status
Decomposed into sub-tasks.

## Goal
A dashboard app that analyzes user patterns (calendar, notes, to-do, screen-time if available) and uses a local Ollama LLM to suggest actionable tasks, improvements, and reminders. Suggestions appear as cards that the user can accept, dismiss, or convert into a to-do item.

## Acceptance Criteria
- [ ] T-020-a: JS app module + wiring
- [ ] T-020-b: CSS module
- [ ] T-020-c: Server API endpoint `/api/ai/suggest`
- [ ] T-020-d: Settings integration (toggle, model selector)

## Data Flow
1. User opens "AI Suggester" app.
2. Frontend POSTs a summary of recent dashboard activity to `/api/ai/suggest`.
3. Server calls `_ollama_generate()` with a crafted prompt.
4. Server returns structured JSON: `{ suggestions: [{ id, title, description, priority, category }] }`.
5. Frontend renders suggestion cards with Accept / Dismiss / Create Task actions.
6. Dismissed IDs stored in `localStorage` (`ncc-ai-suggester-dismissed`).
7. Accept writes a to-do item via the existing To-Do app (or localStorage bridge).

- Optional: T-048 Usage analytics for richer prompts.

## localStorage Keys
- `ncc-ai-suggester-dismissed` — array of dismissed suggestion IDs.
- `ncc-ai-suggester-cache` — last suggestion response (timestamped).
- `ncc-ai-suggester-settings` — `{ enabled: true, model: 'llama3.2', frequency: 'daily' }`
