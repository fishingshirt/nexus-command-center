# T-059 — Hermes Chat Bridge: Replies Never Arrive

## Status
`PENDING`

## Severity
`HIGH`

## Bug Description
The dashboard chat widget can **send** messages to Hermes, but the AI **never replies** in the chat. The user sees:
> "Sent to Hermes. Waiting for a reply…"

But no follow-up message ever appears.

## Root Cause Hypotheses (To Investigate)
1. **Poll endpoint broken** — `GET /api/hermes/poll` calls `_tg_api('getUpdates')` but may fail silently, return nothing, or the message deduplication logic (`lastSeenId`) filters out valid replies.
2. **No reply channel** — Hermes (this AI) is not routed back into the dashboard. When the user sends via Telegram, the reply goes to Telegram DM. The dashboard doesn't see it.
3. **Polling stops too early** — The interval timer gets killed prematurely or the user leaves the page before the poll succeeds.
4. **Auth / token** — `_tg_token()` or `_tg_chat_id()` might fail to resolve, silently dropping the reply logic.

## Reproduction Steps
1. Open dashboard → tap **Hermes Chat**
2. Type a message and send
3. Observe "Sent to Hermes. Waiting for a reply…"
4. Wait indefinitely — **no reply appears**
5. Check Telegram app → message WAS delivered, AI **did** reply there, but dashboard stays empty
