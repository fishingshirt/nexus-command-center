# T-046 — Auto Dark Mode + System Theme Sync

## Goal
The theme engine should automatically switch between light and dark variants based on system preference or time of day, with per-app exceptions.

## Acceptance Criteria
- [ ] System preference detection: `window.matchMedia('(prefers-color-scheme: dark)')` → switch to dark variant of current theme
- [ ] Time-based toggle: optional "sunset to sunrise" auto-switch with user-configured times
- [ ] Per-app theme exception: allow user to force a specific theme for individual apps (e.g. IDE simulator always dark)
- [ ] Smooth transition: CSS `transition` on `--bg`, `--text`, `--accent` variables so switch isn't jarring
- [ ] Settings panel additions:
  - "Follow system theme" toggle
  - "Auto-switch by time" toggle + time pickers
  - "Transition duration" slider (0-1s)
  - Per-app theme overrides
- [ ] Dashboard icon: small moon/sun icon in header when auto-mode is active

## UI Details
- Transition: all color properties transition over 300ms default
- Header indicator: 🌙 when dark active, ☀️ when light active
- Time pickers: native `<input type="time">`

## Tech Notes
- Add `--bg-dark`, `--text-dark`, `--accent-dark` variants to each theme CSS file
- Auto-switcher module listens to `matchMedia` changes and time tick
- If user manually picks a theme, auto-switcher pauses until "reset to auto" is clicked
- Store: `themeMode: 'auto-system' | 'auto-time' | 'manual'`, `darkStart: '20:00'`, `lightStart: '06:00'`
