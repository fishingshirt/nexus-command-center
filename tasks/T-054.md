# T-054: App Launcher Customization

## Status
`PENDING`

## Goal
Let users fully customize the main app menu / launcher: show or hide individual apps, reorder them via drag-and-drop, and reset to defaults. Previously pinned favorite apps remain untouched in their own dock.

## Acceptance Criteria

- [ ] **App Visibility Toggle**
  - Settings panel section: "Visible Apps" with a list of all registered apps.
  - Each row: app icon, name, visibility toggle switch.
  - Hidden apps do NOT appear in the launcher grid or sidebar.
  - Hidden apps still accessible via Global Search (T-041) and via direct URL (`/#app-id`).
  - Favorites bar (bottom dock) is unaffected by visibility toggles — pinned apps stay pinned even if hidden from the main menu.
  - Default: all apps visible on first boot.
  - Hidden apps count shown as a subtle badge on the section header: "8 visible, 4 hidden".

- [ ] **Drag-to-Reorder**
  - In the "Visible Apps" settings list, each row has a drag handle (⋮⋮).
  - Drag to reorder apps — updates the launch order in the main menu grid/sidebar immediately (live sync).
  - New app order persisted to `localStorage` key `ncc-app-order`.
  - On first boot, order falls back to the built-in registration order.
  - "Reset to Default Order" button in the section header.
  - Haptic feedback on drag start / drop (if supported).

- [ ] **Main Menu / Launcher**
  - The launcher grid / sidebar renders ONLY visible apps, in the user-defined order.
  - Empty state: if user hides ALL apps, show "All apps are hidden. Tap the settings gear to show some." with a link to the Apps settings panel.
  - App grid supports the existing drag-and-drop reordering already built for widgets (if any), or use a simple sortable list in settings only.

- [ ] **Favorites Dock** (unchanged but aware)
  - Bottom dock / favorites bar still works exactly as before (T-037).
  - Pinned favorites are NOT affected by the hide/show toggle.
  - Changing app order in settings does NOT change favorites order — favorites have their own `ncc-favorites` array.

- [ ] **Settings Schema Addition**
  ```json
  "appLauncher": {
    "hiddenApps": ["finance", "habit"],
    "appOrder": ["dashboard", "calendar", "news", "notes", "todo", "chat", "settings"]
  }
  ```

## UI Details
- Settings → "App Launcher" section.
- Section header with count, "Reset" link, and optional "Show All" / "Hide All" quick toggles.
- Each app row: icon (24px), app name, toggle switch, drag handle (⋮⋮).
- Toggle switch styled in theme accent color.
- Drag handle is dimmed until hovered; on mobile, long-press activates drag.
- Drag ghost is a translucent card with the app icon and name.

## Tech Notes
- `localStorage` keys:
  - `ncc-app-visibility`: `{ [appId]: boolean }`
  - `ncc-app-order`: `string[]` of app IDs in user order.
- On app registration in `public/js/apps.js`, if the app is not in `ncc-app-order`, append it to the end.
- Re-render launcher on visibility/order change without page reload.
- Keep favorites logic (`ncc-favorites`) completely isolated from launcher customization.

## Files Expected to Touch
- `public/index.html` — settings panel markup.
- `public/js/apps.js` — registration + launcher render using visibility/order.
- `public/js/settings.js` — add "App Launcher" settings section.
- `public/css/apps/launcher.css` — new or update existing launcher CSS.
- `nexus-server.py` — no changes needed (pure client-side feature).

This is a purely client-side feature. No server endpoints needed. Focus on smooth UX—drag should feel native, and visibility changes should reflect instantly in the launcher.
