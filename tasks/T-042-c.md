# T-042-c: Wire Calendar + Todo to Notification System

## Status
`PENDING`

## Goal
Make Calendar and To-Do apps emit notifications for reminders and due dates.

## Acceptance Criteria
- [ ] Calendar: when an event start time is within 15 minutes, emit `notify({ title: 'Upcoming: Event Name', body: 'Starts in 15 min', app: 'calendar', priority: 'high' })`
- [ ] Calendar: auto-dismiss reminder after event start time passes (remove on next poll)
- [ ] Todo: when a task due date is within 24 hours and not completed, emit `notify({ title: 'Task Due Soon', body: task.title, app: 'todo', priority: 'normal' })`
- [ ] Todo: only emit once per task per day (track `ncc-notified-tasks` timestamp map)
- [ ] Polling interval: every 60 seconds via `setInterval` in `app.js` (centralized poller, not per-app)
- [ ] Poller must be cleaned up on `beforeunload` to avoid memory leaks
- [ ] Respect DND hours if implemented (future sub-task)

## Files Expected to Touch
- `public/js/app.js` — add centralized notification poller `startNotificationPoller()`
- `public/js/apps/calendar.js` — expose `getUpcomingEvents(minutes)` helper
- `public/js/apps/todo.js` — expose `getDueSoonTasks(hours)` helper

## LocalStorage Keys
- `ncc-notified-tasks` — `{ [taskId]: '2026-05-10' }` (date string to dedupe)
- `ncc-notified-events` — `{ [eventId]: timestamp }` (timestamp to dedupe)

## Test
- Create calendar event 10 min from now → wait for poll → notification appears
- Mark task due tomorrow → wait for poll → notification appears once
- Complete task → no further notifications for that task
