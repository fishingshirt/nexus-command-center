# T-019-c: Finance Tracker — Server API Endpoints (Optional v1 Server Backup)

## Goal
Add server endpoints for finance data backup/restore so data survives browser clear.

## Acceptance Criteria
1. In `scripts/nexus-server.py`, add endpoints:
   - `GET /api/finance/read` → return JSON `{transactions: [...]}` from `data/finance.json` if exists, else `{transactions: []}`
   - `POST /api/finance/write` → body `{transactions: [...]}`, write to `data/finance.json`, return `{ok: true}`
2. Ensure `data/` directory exists on server startup.
3. In `public/js/apps/finance-tracker.js`, add optional server sync:
   - On `initApp`, try `fetch('/api/finance/read')`. If 200, merge server data (server wins on ID conflict)
   - On every add/delete, debounced `fetch('/api/finance/write', {method:'POST', body: ...})`
   - If server unreachable, fall back to localStorage only (no error toast)
4. Data file `data/finance.json` must be `.gitignore`d so user data is never committed.

## Files Expected to Touch
- `scripts/nexus-server.py` (new endpoints)
- `.gitignore` (add `data/finance.json`)
- `public/js/apps/finance-tracker.js` (server sync block — can be added after T-019-a)
