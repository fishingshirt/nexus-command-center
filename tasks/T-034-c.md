# T-034-c — Google Calendar Conflict Resolution & Offline Queue

## Goal
Handle the case where an event is modified on both the dashboard and Google Calendar since last sync. Show a visual diff in the UI and let the user pick which version wins. Also queue outbound mutations in `localStorage` while offline and auto-flush when connection returns.

## Acceptance Criteria
- [ ] Each local event stores `lastSyncedAt` and `lastModifiedAt` timestamps.
- [ ] Inbound sync (`T-034-a` pull) compares `updated` time from Google with local `lastModifiedAt`.
- [ ] If both changed since `lastSyncedAt`, mark event with `conflict: true` and show a yellow badge in calendar day view.
- [ ] Clicking conflict badge opens a mini diff panel: left = local version, right = Google version, two buttons: "Keep Mine" / "Use Google".
- [ ] Choosing "Keep Mine" → triggers `events.patch` to Google with local data.
- [ ] Choosing "Use Google" → overwrites local event with Google data.
- [ ] Offline queue: `localStorage` key `ncc-calendar-sync-queue` stores array of `{ action: 'insert'|'patch'|'delete', payload, retryCount }`.
- [ ] On `nexusOnline` event, iterate queue and send each item to `/api/calendar/events`; remove from queue on success, increment `retryCount` on failure; stop retrying after 3 attempts.
- [ ] Sync status badge shows `synced` only after outbound mutations complete successfully.

## Files to Touch
