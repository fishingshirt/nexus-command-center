# T-019-b: Finance Tracker — CSS Module + Theme Integration

## Goal
Create the Finance Tracker stylesheet and import it into the theme system.

## Acceptance Criteria
1. Create `public/css/apps/finance-tracker.css` with:
   - `.finance-tracker` container layout (mobile-first, max-width 800px)
   - Form grid: amount + type toggle in one row, category + date in next, note full width
   - Transaction list rows with alternating background using `var(--bg-card-alt)`
   - Monthly summary bar: 3 equal columns, bold numbers
   - Category breakdown bar: flex row of segments, each with unique CSS var color
   - Delete button is icon-only (trash can emoji or Unicode) with `aria-label`
   - Responsive: at <768px stack form fields vertically
   - Dark mode compatible: uses `var(--bg-body)`, `var(--text-main)`, `var(--text-muted)`
   - No hardcoded hex colors; all via CSS custom properties or theme variables
2. Import in `public/css/base.css` or the active theme CSS (whichever the project uses for per-app imports):
   - Add `@import url('apps/finance-tracker.css');` if that is the pattern
   - Verify the project pattern by looking at how other app CSS files are imported.

## Files Expected to Touch
- `public/css/apps/finance-tracker.css` (new)
- `public/css/base.css` or theme file (import line)

## Dependencies
T-019-a (JS must exist for selectors to work)
