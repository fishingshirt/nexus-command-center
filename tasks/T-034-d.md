# T-034-d — Agent Calendar Command Endpoint

## Status: DONE

Added `POST /api/calendar/agent/add` and `GET /api/calendar/agent/poll` endpoints in `nexus-server.py`, plus frontend polling in `public/js/apps/gcal-sync.js`.

## What changed
- **Server (`scripts/nexus-server.py`)**
  - `_parse_agent_event()` natural-language parser extracts title, date, start/end from strings like *"Dentist appointment tomorrow at 3pm"*.
  - `POST /api/calendar/agent/add` — accepts `{ text, force? }`; parses event; either queues or (if `force:true`) immediately inserts into Google Calendar.
  - `GET /api/calendar/agent/poll` — returns queued events and clears the queue (acknowledged).
- **Frontend (`public/js/apps/gcal-sync.js`)**
  - `pollAgentEvents()` called on startup and every `window.focus`.
  - Pulls queued agent events, deduplicates, appends to `ncc-calendar-events`, queues for outbound sync.
- **Agent notes — not behind PIN gate** because the dashboard itself is already behind the TEMP PIN cookie (`nexus_pin`).

## Testing
- Run `POST /api/calendar/agent/add` with `{ "text": "Call Alice next Tuesday at 2pm" }`
- Open dashboard → it should pull the event on focus and toast "1 event added from agent".
