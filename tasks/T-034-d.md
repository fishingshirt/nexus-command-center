# T-034-d — Agent Calendar Command Endpoint

## Goal
Allow the user (via Telegram or any external agent) to tell Nexus: *"Add event X to my calendar"*. The dashboard exposes a POST endpoint that accepts a natural-language event description, parses it into structured data, creates a local calendar event, and syncs it to Google Calendar via the existing outbound engine (`T-034-b`).

## Acceptance Criteria
- [ ] New `POST /api/calendar/agent/add` endpoint in `nexus-server.py`.
- [ ] Request body: `{ text: "Dentist appointment tomorrow at 3pm", force?: boolean }`.
- [ ] Server parses `text` with a lightweight heuristic (regex + `dateutil.parser` if available; fallback to naive split):
  - Extract date phrase (`tomorrow`, `next Monday`, `May 15`, etc.).
  - Extract time phrase (`3pm`, `14:00`, etc.).
  - Remainder = title.
- [ ] Parsed event written to server-side queue file `~/.hermes/nexus-agent-calendar-queue.json`.
- [ ] Dashboard frontend polls `/api/calendar/agent/poll` on startup and whenever it gains focus; pulls new events into `localStorage` under `ncc-calendar-events`, then calls the outbound sync (`POST /api/calendar/events`) to push to Google Calendar.
- [ ] If `force: true`, the server immediately attempts the Google Calendar insert (requires token from `T-034-a`) and returns `{ ok:true, event }` or `{ ok:false, error }`.
- [ ] Endpoint is PIN-protected or behind `TEMP_PIN` cookie to prevent unauthorized writes.

## Files to Touch
- `scripts/nexus-server.py` — add `/api/calendar/agent/add` and `/api/calendar/agent/poll`.
- `public/js/apps/calendar.js` — add polling for agent events on init and on window focus.
- `public/js/apps/gcal-sync.js` — hook agent poll into outbound sync pipeline.

## API Contract
- **POST** `/api/calendar/agent/add`
  - Body: `{ text: string, force?: boolean }`
  - Response: `{ ok: true, event: { id, title, date, start, end } }` or `{ ok: false, error }`
- **GET** `/api/calendar/agent/poll`
  - Response: `{ events: [...], count: number }`
  - Events are removed from server queue after being polled once (acknowledged).

## Error Handling
- If `dateutil` not installed, use regex fallback (`tomorrow` = today+1, `next Monday` = next weekday).
- If parsing fails, return `{ ok:false, error:'Could not parse date/time' }`.
- If no OAuth token, return `{ ok:false, error:'Google Calendar not connected' }`.
