# T-034-a — Google Calendar OAuth Token Flow (frontend + backend)

## Goal
Replace the read-only API-key sync with a proper OAuth 2.0 authorization-code flow that obtains a refresh token for full `https://www.googleapis.com/auth/calendar` scope (read/write). The Python server must exchange the authorization code for tokens and securely store the refresh token server-side so the dashboard can make outbound mutations.

## Acceptance Criteria
- [ ] Add a "Connect Google Calendar" button in Settings → Calendar Sync that starts `https://accounts.google.com/o/oauth2/v2/auth?...` flow.
- [ ] Redirect URI is `http://localhost:8080/api/calendar/oauth/callback` handled by `nexus-server.py`.
- [ ] Server exchanges code for `access_token` + `refresh_token` via `https://oauth2.googleapis.com/token`.
- [ ] Refresh token stored in `~/.hermes/nexus-gcal-tokens.json` (never returned to client).
- [ ] New `api_client_secret` field in Settings (or server reads from `~/.hermes/.env` `GOOGLE_CLIENT_SECRET`).
- [ ] After auth, Settings badge shows "Re-authorize" if scope is still read-only; click upgrades scope.
- [ ] UI removes API key input (deprecated); shows connected account email if available.

## Files to Touch
- `public/index.html` — Settings panel: replace API key input with Connect button, add `clientId` + `clientSecret` fields.
- `public/js/app.js` — `initCalendarSync()` wiring for OAuth button and status.
- `public/js/apps/gcal-sync.js` — remove API-key logic; use `/api/calendar/sync` endpoint instead.
- `scripts/nexus-server.py` — add `do_GET` handlers for `/api/calendar/oauth/start`, `/api/calendar/oauth/callback`, and token refresh helper.

## `localStorage` Keys
- `ncc-settings.calendarSync` shape: `{ clientId, connected: boolean, lastSyncedAt, status }`
- No `apiKey` field after migration; old key ignored.

## Server-Side Storage
- `~/.hermes/nexus-gcal-tokens.json`:
  ```json
  {
    "refresh_token": "...",
    "access_token": "...",
    "expires_at": 1715300000,
    "scope": "https://www.googleapis.com/auth/calendar",
    "token_type": "Bearer"
  }
  ```

## Error Handling
- If token exchange fails, show error toast: "Google auth failed — check client secret."
- If user revokes token server-side (deletes `nexus-gcal-tokens.json`), UI falls back to "Not linked".
- Server fails open: `fetch` to Google with missing token returns `{ offline: true }`, UI shows "Not linked".
