# T-031-c: Cloud Vault — Server API Endpoints + AES Encryption

## Goal
Add server endpoints and encrypt files at rest.

## Acceptance Criteria
1. In `scripts/nexus-server.py`, add:
   - `POST /api/vault/upload` — multipart file upload. Save to `data/vault/` (preserve folder path if provided). Generate file ID (ulid or uuid). Return `{ok:true, id, filename, size}`
   - `GET /api/vault/list?folder=` — return JSON `{items:[{id, filename, folder, size, ts, mime}]}`
   - `GET /api/vault/download?id=` — return file bytes with correct `Content-Type`
   - `POST /api/vault/delete` — body `{id}`, remove file + metadata
   - `POST /api/vault/move` — body `{id, newFolder}`
2. Encrypt at rest:
   - When saving uploaded file, encrypt with AES-256-GCM using a key derived from `ENCRYPTION_KEY` env var (fallback to a fixed derived key from server config)
   - Store ciphertext + IV + tag in a file; keep metadata (id, filename, folder, size, ts, mime) in a separate JSON index `data/vault-index.json`
   - On download, decrypt and stream response
3. On server startup, ensure `data/vault/` exists.
4. Add `data/vault/` and `data/vault-index.json` to `.gitignore` so user files are never committed.

## Files Expected to Touch
- `scripts/nexus-server.py` (new endpoints + encryption helpers)
- `.gitignore` (add vault paths)
- `public/js/apps/vault.js` (optional: adjust fetch paths if needed)
