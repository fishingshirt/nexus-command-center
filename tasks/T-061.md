# T-061 — Wishlist App

> **Priority:** MEDIUM  
> **Status:** PENDING

---

## Overview

A standalone wishlist app where the user can save items they want to buy, track, or remember. Each item includes a link, optional image (fetched or uploaded), price, notes, priority, tags, and status. The app supports categorization, search, and sorting.

---

## Functional Requirements

### Item Schema
| Field | Type | Required | Default |
|-------|------|----------|---------|
| `id` | UUID | auto | — |
| `title` | string | yes | — |
| `url` | string (URL) | no | — |
| `image` | base64 / URL | no | placeholder |
| `price` | number | no | — |
| `currency` | enum | no | USD |
| `priority` | enum | no | Normal |
| `status` | enum | no | Want |
| `tags` | string[] | no | [] |
| `notes` | string | no | " " |
| `created` | ISO date | auto | now |
| `updated` | ISO date | auto | now |

**Priority enum:** `Low | Normal | High | Must Buy`  
**Status enum:** `Want | Watching | Purchased | Archived`  
**Currency enum:** `USD | EUR | GBP | CAD | JPY | Other`

### Features
1. **Add Item** — modal form with title, URL, price, currency, priority, tags, notes. Optional image URL input.
2. **Auto-fetch** — pasting a URL triggers background fetch (via server-side scraping or dashboard API endpoint) to populate title, image, and price if available. Falls back gracefully on failure.
3. **List View** — grid of cards with image thumbnail, title, price badge, priority color strip, status pill.
4. **Detail View** — click card opens full detail overlay with all fields editable inline.
5. **Search** — real-time filter by title, notes, or tags.
6. **Sort** — by price, priority, status, date created, date updated.
7. **Filter** — by status, priority, tag.
8. **Status transitions** — one-click actions: `Move to Watching`, `Mark Purchased`, `Archive`. Archived items hidden by default (toggle).
9. **Quick notes** — inline note editing from detail view without opening full edit modal.
10. **Bulk actions** — multi-select checkboxes for batch archive / delete / change status.
11. **Share / Export** — copy link to clipboard, or export selected items to markdown list.

## UI Requirements

- Match existing dashboard app shell pattern (`app-shell` iframe, `index.html` + `style.css` + `script.js`)
- Use dark-first theme variables (jarvis palette default)
- Mobile-first card grid: 1 column on mobile, 2 on tablet, 3 on desktop
- Image lazy loading for performance
- Skeleton loaders while fetching link metadata
- Empty state illustration with CTA to add first item

---

## Technical Requirements

- Persist to `localStorage` under key `nexus_wishlist_v1`
- Data adapter pattern matching `storage_adapter.js` interface
- Image fetching proxy: dashboard server API endpoint `/api/fetch-link` to avoid CORS when scraping metadata
- Graceful degradation: if server proxy unavailable, allow manual entry only
- No external APIs required for core functionality
