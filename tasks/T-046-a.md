# T-046-a: Auto Dark Mode — CSS Variable Transition + matchMedia Listener

## Goal
Enable smooth theme switching based on system preference and time, using CSS variable transitions.

## Acceptance Criteria
1. Update every theme CSS file (`public/css/themes/*.css`) to include dark overrides:
   - Add `body[data-theme-mode="dark"]` selectors that override `--bg`, `--text`, `--accent`, `--border`, `--card-bg`, etc.
   - Ensure there is a `transition: background-color 0.3s, color 0.3s, border-color 0.3s;` on `body` or `:root`
2. Create `public/js/theme-autoswitch.js` with:
   - `matchMedia('(prefers-color-scheme: dark)')` listener → toggles `body.dataset.themeMode = 'dark' | 'light'`
   - Time-based check: if user enabled auto-time, compare current time to `darkStart` / `lightStart` from settings
   - If user manually picks a theme, pause auto-switcher until reset
   - Store `themeMode` (auto-system | auto-time | manual) in `localStorage` key `ncc-theme-mode`
3. Wire in `public/index.html`: add `<script type="module" src="js/theme-autoswitch.js"></script>` before `</body>`
4. Header indicator: add small sun/moon icon in header that updates when auto-mode switches

## Files Expected to Touch
- `public/css/themes/*.css` (dark overrides)
- `public/js/theme-autoswitch.js` (new)
- `public/index.html` (script tag + header icon)
- `public/js/app.js` (call themeAutoswitch init)

## Dependencies
None
