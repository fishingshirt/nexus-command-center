# T-034-b — Google Calendar Outbound Sync Engine (POST /api/calendar/events)

## Goal
Once `T-034-a` provides a valid OAuth access token, enable dashboard events to propagate **to** Google Calendar. Creating, editing, or deleting an event in `calendar.js` must call the Python backend, which forwards the mutation to Google Calendar API v3 (`events.insert`, `events.patch`, `events.delete`). The local event must store the returned `googleEventId` for stable mapping.

## Acceptance Criteria
- [ ] `calendar.js` `saveEvent()` after persisting to `localStorage` calls `fetch('/api/calendar/events', { method:'POST', body: JSON.stringify(event) })`.
- [ ] `calendar.js` `deleteEvent()` calls `fetch('/api/calendar/events', { method:'DELETE', body: JSON.stringify({id}) })`.
- [ ] If the event already has a `gcalId`, backend uses `events.patch` instead of `events.insert`.
- [ ] On success, backend returns `{ ok:true, gcalId }` and frontend updates the local event with that `gcalId`.
- [ ] On failure (network or token), backend returns `{ ok:false, error }` and frontend shows toast; mutation stays in localStorage (offline queue handled in T-034-c).
- [ ] Backend refreshes access token automatically if expired (using stored refresh token from `T-034-a`).
- [ ] No `console.error` in new JS; use `toast('Calendar sync failed', 'error')`.

## Files to Touch
- `public/js/apps/calendar.js` — modify `saveEvent()` and `deleteEvent()` to POST/DELETE to backend.
- `public/js/apps/gcal-sync.js` — remove direct `fetch` to `googleapis.com`; rely on backend proxy.
- `scripts/nexus-server.py` — add `do_POST` route `/api/calendar/events` with insert/patch/delete logic and token refresh.

## API Contract
- **POST** `/api/calendar/events`
  - Body: `{ id, title, date, start, end, category, recurrence, description, gcalId? }`
  - Response: `{ ok: true, gcalId }` or `{ ok: false, error }`
- **DELETE** `/api/calendar/events`
  - Body: `{ id, gcalId }`
  - Response: `{ ok: true }` or `{ ok: false, error }`

## Google API Mapping
- `date` → `start.date` (all-day) or `start.dateTime` + timezone.
- `start`/`end` (HH:MM) → ISO date-time on the event date.
- `recurrence` (daily/weekly/monthly/yearly) → `RRULE:FREQ=DAILY;...`
- `category` → `colorId` (optional, not critical).
