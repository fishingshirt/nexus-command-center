# T-060-d — Dashboard Quality Status Panel

## Status
`PENDING`

## Goal
Add a quality status panel to the dashboard so the user (and agent) can see the current QC/QA pipeline at a glance.

## Acceptance Criteria
- [ ] New server endpoint `GET /api/quality/queue` returns the current queue JSON
- [ ] New server endpoint `GET /api/quality/result/:id` returns a specific QC/QA result
- [ ] New server endpoint `POST /api/quality/approve` allows the agent to submit sign-off (auth: check cookie or API key)
- [ ] Add a small widget to the Agent Stats page showing:
  - Queue count by status (QC-PENDING, QC-PASSED, QA-PENDING, QA-PASSED, BACKLOG)
  - Last audit timestamp
  - Pass/fail rate of last 10 audits
- [ ] Color coding: green = pass, red = fail, yellow = in progress, gray = backlog
- [ ] Click to expand: show last 5 items in queue with file names and status

## UI Details
- Use existing card/badge styling from base.css
- Keep it compact — max height 200px, scrollable
- Mobile: show only counts, tap to expand

## Agent Note
This panel is for visibility only — it does not replace the actual QC/QA pipeline. The agent still runs scripts directly.
