# T-049 — File Attachments for Notes & Calendar

## Goal
Allow users to attach files (images, PDFs, documents) to notes and calendar events. Stored locally, viewable inline.

## Acceptance Criteria
- [ ] Drag-and-drop onto note editor or event modal → saves file
- [ ] File picker fallback: standard `<input type="file">`
- [ ] Supported types: images (png, jpg, webp, gif), PDFs, text files
- [ ] Inline preview: images show thumbnail, PDFs show filename + download link
- [ ] Per-item storage: file metadata in app data, blob stored in IndexedDB (localStorage too small)
- [ ] File size limit: 5MB per file, max 50MB total per app
- [ ] Download: click attachment to download original file
- [ ] Delete: remove attachment from item without deleting the item
- [ ] Agent command: "attach the PDF to my grocery note" or "show attachments for event team lunch"

## UI Details
- Attachment strip: horizontal row of thumbnails below note/event body
- Image thumbnail: 80x80px, rounded corners, object-fit cover
- PDF icon: generic document icon with filename
- Drag overlay: dashed border highlight when dragging over editor
- Upload progress: thin progress bar for large files

## Tech Notes
- IndexedDB wrapper: simple `idb_keyval` pattern (or raw IDB, no library)
- Store blobs under key `attachments/{noteId}/{fileHash}`
- Hash filenames with `crypto.subtle.digest('SHA-256', blob)` to dedupe
- Fallback if IDB unavailable: warn user "File storage unavailable in this browser"
- Clean up orphaned blobs on boot (orphan scan)

## Agent Note
This touches Notes and Calendar app data schemas. Add `attachments: []` array to both. The IndexedDB layer should be a shared utility (`public/js/storage.js`) so other apps can reuse it later.
