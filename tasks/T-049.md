# T-049 — File Attachments for Notes & Calendar

## Status
✅ DONE

## Goal
Allow users to attach files (images, PDFs, documents) to notes and calendar events. Stored locally, viewable inline.

## Acceptance Criteria
- [x] Drag-and-drop onto note editor or event modal → saves file
- [x] File picker fallback: standard `<input type="file">`
- [x] Supported types: images (png, jpg, webp, gif), PDFs, text files
- [x] Inline preview: images show thumbnail, PDFs show filename + download link
- [x] Per-item storage: file metadata in app data, blob stored in IndexedDB (localStorage too small)
- [x] File size limit: 5MB per file, max 50MB total per app
- [x] Download: click attachment to download original file
- [x] Delete: remove attachment from item without deleting the item
- [x] Agent command: "attach the PDF to my grocery note" or "show attachments for event team lunch"

## UI Details
- Attachment strip: horizontal row of thumbnails below note/event body
- Image thumbnail: 80x80px, rounded corners, object-fit cover
- PDF icon: generic document icon with filename
- Drag overlay: dashed border highlight when dragging over editor
- Upload progress: thin progress bar for large files

## Tech Notes
- IndexedDB wrapper: simple `idb_keyval` pattern (or raw IDB, no library)
- Store blobs under key `attachments/{noteId}/{fileHash}`
- Hash filenames with `crypto.subtle.digest('SHA-256', blob)` to dedupe
- Fallback if IDB unavailable: warn user "File storage unavailable in this browser"
- Clean up orphaned blobs on boot (orphan scan)
