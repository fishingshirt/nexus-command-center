# T-062 — Recipe App

> **Priority:** MEDIUM  
> **Status:** PENDING  
> **Created:** 2026-05-10

---

## Overview

A personal cookbook app for saving, organizing, and rating recipes. Each recipe includes ingredients, instructions, prep/cook time, servings, tags, photos, and user notes/ratings. Supports categorization by meal type, cuisine, and dietary tags.

---

## Functional Requirements

**Meal type enum:** `Breakfast | Lunch | Dinner | Snack | Dessert | Drink`  
**Dietary enum:** `Vegetarian | Vegan | Gluten-Free | Dairy-Free | Keto | Paleo | Halal | Kosher | Low-Sodium | Nut-Free`

### Features
1. **Add Recipe** — multi-step form:
   - Step 1: Basic info (title, description, images, source URL)
   - Step 2: Time & servings (prep, cook, servings, difficulty)
   - Step 3: Ingredients (dynamic rows: name, amount, unit, optional checkbox)
   - Step 4: Instructions (numbered steps, drag-to-reorder)
   - Step 5: Tags, cuisine, dietary, meal type
2. **Auto-parse** — pasting a source URL attempts to scrape recipe data (title, ingredients, instructions, image) via `/api/fetch-recipe` proxy. User can review and edit before saving.
3. **List View** — card grid with cover image, title, rating stars, cook time badge, difficulty chip, favorite heart icon.
4. **Search & Filter** — by title, ingredient, tag, cuisine, dietary, meal type, difficulty, cook time range.
5. **Sort** — by rating, cook time, recently added, recently cooked, title alphabetical.
6. **Detail View** — full recipe page with:
   - Image carousel (swipe on mobile)
   - Servings scaler (+- buttons multiply ingredient amounts)
   - Step-by-step cooking mode (fullscreen, one step at a time, timer per step)
   - Quick action: `I cooked this!` button logs date to history
   - Inline note editing
7. **Rate & Review** — 5-star rating per recipe + free-text notes section visible in detail view.
8. **Cook History** — logs each time user hits "I cooked this!" with date and optional quick note. Viewable as a timeline on the recipe detail page.
9. **Favorites** — toggle heart on any card/recipe. Filter to show favorites only.
10. **Categories** — sidebar or top filter chips for cuisine, meal type, dietary.
11. **Import / Export** — export single recipe or all recipes to JSON. Import from JSON to restore/transfer.
12. **Shopping List Generator** — from recipe detail or multi-select: generate a combined shopping list of ingredients, groupable by recipe or by category (produce, pantry, etc.).

## UI Requirements

- Match existing dashboard app shell pattern (`app-shell` iframe, `index.html` + `style.css` + `script.js`)
- Dark-first theme variables (jarvis palette default)
- Mobile-first card grid: 1 column on mobile, 2 on tablet, 3+ on desktop
- Ingredient rows use compact layout (name | amount unit | optional)
- Step-by-step cooking mode: high contrast, large text, touch-friendly prev/next buttons
- Image upload: drag-and-drop or file picker, store as base64 (limit 500KB per image)
- Skeleton loaders for auto-fetched recipes
- Empty state with CTA

---

## Technical Requirements

- Persist to `localStorage` under key `nexus_recipes_v1`
- Data adapter pattern matching `storage_adapter.js` interface
- Recipe scraping proxy: dashboard server API endpoint `/api/fetch-recipe` for auto-parse (avoids CORS)
- Image compression: client-side canvas resize before storing base64 (max 800px wide)
- No external APIs required for core functionality

## Acceptance Criteria

- [ ] User can create a recipe with title, ingredients, instructions, and images
- [ ] Ingredients display with amounts that scale when servings change
- [ ] Recipes are searchable, filterable, and sortable
- [ ] User can rate any recipe 1-5 stars and add notes
- [ ] "I cooked this!" logs date to cook history
- [ ] Favorites toggle persists
- [ ] All data persists across reloads
- [ ] Works offline after first load
- [ ] Mobile layout is usable (readable steps, thumb-friendly buttons)
