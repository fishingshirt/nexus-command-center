# T-039 — Multi-City Weather Manager

## Goal
Upgrade the weather widget so the user can save, manage, and view multiple cities/places. The main dashboard should show a quick-glance carousel or grid of saved locations.

## Acceptance Criteria
- [ ] **Add Location** — search by city name or ZIP/postal code, geocode via Open-Meteo or similar, preview before saving
- [ ] **Saved Locations List** — stored in `localStorage` under `ncc-weather-locations`; reorderable via drag-and-drop
- [ ] **Main Dashboard View** — compact cards for each saved city showing: current temp, condition icon, high/low; tap to expand full forecast
- [ ] **Default / Home Location** — one location marked as "home" that shows expanded by default on the weather app tab
- [ ] **Delete / Edit** — swipe or long-press to delete; edit to rename display label
- [ ] **Auto-refresh** — all saved locations refresh every 15 minutes (or on dashboard focus)
- [ ] **Agent Integration** — user can tell agent "add Miami to my weather" or "remove Tokyo from weather" and agent updates the saved locations list

## UI Details
- **Dashboard mini-view:** horizontal scroll row of location pills (`🌤️ NYC 72°`, `☀️ Miami 88°`, `⛈️ London 58°`)
- **Weather app full-view:** vertical stack of expanded location cards with 5-day forecast per city
- **Settings > Weather:** manage list, set home location, toggle units (°F/°C), set refresh interval

## Tech Notes
- Reuse existing Open-Meteo geocoding + forecast endpoints.
- Store as array: `[{id, name, lat, lon, country, isHome, addedAt}]`.
- Max saved locations: 10 (to keep API calls reasonable).
