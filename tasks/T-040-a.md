# T-040-a: Dashboard Widget Grid — Core Engine + Default Layout

## Goal
Build the widget grid engine and wire it into the dashboard so apps can register widgets.

## Acceptance Criteria
1. Create `public/js/widgets/grid.js` with:
   - `WidgetGrid` class: manages a CSS-grid container, persists layout to `localStorage` key `ncc-dashboard-widgets`
   - `registerWidget(type, factoryFn)` — apps call this to expose widgets
   - `renderLayout()` — reads saved layout or uses default, creates cards
   - `enterEditMode()` / `exitEditMode()` — toggles drag handles and dim overlay
   - Drag-and-drop reorder via HTML5 Drag API
   - Default layout: `[{type:'weather'},{type:'calendar-today'},{type:'todo-count'},{type:'agent-stats'}]`
   - Auto-refresh every 60s per widget
2. Wire in `public/index.html`:
   - Add `<div id="widget-grid" class="widget-grid"></div>` inside `#view-dashboard`
   - Add `<script type="module" src="js/widgets/grid.js"></script>` before `</body>`
3. Update `public/js/app.js`:
   - Import `WidgetGrid` and call `new WidgetGrid(document.getElementById('widget-grid'))`
   - After `initApp()`, call `widgetGrid.renderLayout()` and start auto-refresh interval

## Files Expected to Touch
- `public/js/widgets/grid.js` (new)
- `public/index.html` (grid container + script tag)
- `public/js/app.js` (import + init)
- `public/js/apps/weather.js`, `calendar.js`, `todo.js` — add `registerWidget()` calls (example stubs)

## Dependencies
None
