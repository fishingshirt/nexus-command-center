# T-047 — Keyboard Shortcuts / Hotkey System

## Goal
Power-user keyboard navigation across the entire dashboard. Every app and action reachable without a mouse.

## Acceptance Criteria
- [ ] Global shortcuts:
  - `G` then `C` = Go to Calendar
  - `G` then `N` = Go to Notes
  - `G` then `T` = Go to To-Do
  - `/` = Open Global Search
  - `?` = Show shortcuts cheat sheet overlay
  - `Esc` = Close any modal/overlay/app
  - `Ctrl+Shift+W` = Enter Work Simulator (if T-027 built)
- [ ] Within-app shortcuts:
  - Calendar: `N` = New event, `D/W/M` = switch view
  - Notes: `N` = New note, `S` = Save, `F` = Focus search
  - To-Do: `N` = New task, `1/2/3/4` = set priority
- [ ] Shortcuts panel: overlay with all shortcuts, searchable, grouped by context
- [ ] Disable shortcuts when typing in form inputs (check `document.activeElement`)
- [ ] Settings: enable/disable shortcuts globally
- [ ] Visual hint: first-time user sees a toast "Press ? for shortcuts"

## UI Details
- Cheat sheet: modal overlay, two-column grid, section headers (Global, Calendar, Notes, etc.)
- Key display: styled `<kbd>` tags, monospace font
- Discovery: small shortcut hints next to buttons (desktop only, hidden on mobile)

## Tech Notes
- Listener: single `document.addEventListener('keydown', handler)` at app level
- Sequence detection: track last key + timestamp, if < 800ms between keys treat as chord (e.g. `G` → `C`)
- Input guard: `if (tag === 'INPUT' || tag === 'TEXTAREA' || isContentEditable) return`
- Store: `ncc-shortcuts-enabled: true`

## Agent Note
Non-breaking additive feature. Can be built incrementally — start with global shortcuts (`/`, `?`, `Esc`, `G` chords), then add per-app binds. Don't refactor existing click handlers, just add keyboard equivalents.
