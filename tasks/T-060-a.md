# T-060-a — QC Auto Gate: Bug Prevention + Logic & Runtime Validator

## Status
`PENDING`

## Goal
Build the Stage 1 automated quality check script (`scripts/nexus-qc.py`). The **primary purpose of QC is to stop bugs before they ever reach the live dashboard**. QC is not just syntax—it runs logic analysis, functional dry-runs, and regression checks so broken code never hits the main page.

## Philosophy
- **Bugs must not reach production.** QC is the wall.
- If QC passes and a bug still slips through, the QC rules are tightened in the next cycle.
- QC catches mechanical bugs so QA can focus on nuanced logic, UX, and edge cases.

## Acceptance Criteria

### 1. Syntax Validation
- [ ] `check_syntax_js(dir)` — validates all `.js` files via `node --check` or `acorn.parse`. Returns pass/fail per file with exact line numbers.
- [ ] `check_syntax_css(dir)` — regex/parser to catch unclosed braces, missing semicolons on rule endings, malformed `@media` or `@keyframes`
- [ ] `check_html_wellformed(dir)` — validates `index.html` has balanced tags, no duplicate IDs, critical container IDs exist
- [ ] `check_json_valid(dir)` — all `tasks/*.md`, `manifest.json`, `WHITEBOARD.md` must be parseable

### 2. Cross-Reference Validation
- [ ] `check_cross_reference(html_path, js_dir, css_dir)` — every `getElementById('X')` has matching `id="X"`, every `querySelector('.X')` has matching `.X` in CSS
- [ ] `check_app_registry()` — every `data-app="app"` has a matching `#view-app` and file in `public/js/apps/`
- [ ] `check_server_routes(server_path, js_dir)` — every `/api/` string in JS exists in `nexus-server.py` with matching HTTP method
- [ ] `check_task_consistency()` — every task row in `WHITEBOARD.md` has a `tasks/T-XXX.md` file and vice versa

### 3. Logic & Code Quality Analysis (Bug Prevention)
- [ ] `check_logic_js(files)` — static analysis for common logic bugs:
  - Missing `await` on async calls (e.g., `fetch(...)` not awaited)
  - `null`/`undefined` dereference risks (e.g., `x.y` where `x` may be null)
  - Variable shadowing in nested functions
  - Unused variables that may indicate incomplete code
  - `==` instead of `===` in type-sensitive contexts
  - Loop boundaries that look off-by-one (e.g., `<= array.length`)
  - Event listeners added without removal (memory leak risk)
  - `setInterval` without `clearInterval` assignment
- [ ] `check_api_contracts(js_dir, server_path)` — verify the request shapes in JS match what `nexus-server.py` expects (e.g., `POST /api/notes/save` sends `{note: string}` and server reads `note` key)

### 4. Functional Dry-Run & Regression Test
- [ ] `check_runtime_js(files)` — load changed JS files in a mock browser environment (jsdom-lite via Node or minimal `vm` context with mocked `document`, `localStorage`, `fetch`) to catch runtime ReferenceError/TypeError on load
- [ ] `check_regression_smoke()` — run a fast smoke test:
  - Start `nexus-server.py` on a temp port, hit every `/api/` route with basic payloads, verify `200 OK` and JSON shape
  - Run existing `validate-js.js` on all JS files
  - Ensure `index.html` is servable and has no broken resource links

### 5. Security Scan
- [ ] `check_security_scan(files)` — hardcoded secrets/tokens, suspicious `eval/innerHTML`, unescaped user inputs, XSS vectors

### 6. Output & Integration
- [ ] Writes result JSON to `~/.hermes/nexus-quality-queue.json`
- [ ] Exit code 0 = pass, exit code 1 = fail (for CI/githook integration)
- [ ] Logs every check result to `~/.hermes/nexus-audit-log.ndjson` (append-only, never deleted)

## Output Format
```json
{
  "id": "QC-001",
  "timestamp": "2026-05-10T12:00:00Z",
  "commit": "HEAD",
  "files_changed": ["public/js/apps/weather.js"],
  "checks": [
    {"check": "syntax-js", "status": "pass", "detail": "15 files OK"},
    {"check": "logic-js", "status": "FAIL", "detail": "weather.js:187 — fetch() result not awaited, may return Promise instead of data"},
    {"check": "crossref-ids", "status": "pass", "detail": "All 47 getElementById calls validated"},
    {"check": "runtime-js", "status": "pass", "detail": "Mock load OK, no ReferenceError"},
    {"check": "regression-smoke", "status": "pass", "detail": "All 12 API routes returned 200"}
  ],
  "overall": "FAIL",
  "next_action": "BLOCKED — fix logic bug before any human review"
}
```

## Tests
- Run against current repo → should catch any existing logic, syntax, or cross-reference issues
- Run with intentionally broken JS (missing await, bad loop) → must fail
- Run with valid code → must pass
- Temporarily break an API route in server → `regression-smoke` must flag it

## Agent Note
Keep the script self-contained. Use only Python standard library + `subprocess` for Node calls. Do NOT add npm dependencies. The goal is **zero bugs on main** — err on the side of flagging potential issues. If in doubt, FAIL the check and let QA decide.
