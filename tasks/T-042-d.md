# T-042-d: Browser Push + Agent Integration + Settings

## Status
`PENDING`

## Goal
Add browser push notifications, agent POST integration, and user preference settings.

## Acceptance Criteria
- [ ] Settings section: "Notifications" in `view-settings`
  - Toggle: Browser push notifications (default off)
  - Toggle: Sound on new notification (default on)
  - Toggle per app: Calendar, To-Do, Agent, System (default all on)
  - DND hours: start / end time inputs (default 22:00–07:00)
- [ ] On enabling browser push for first time: call `Notification.requestPermission()`
  - If denied, show toast "Push permission denied — check browser settings"
- [ ] Agent can POST to existing `/api/agent/notify` endpoint (already in server)
  - Frontend polls `/api/agent/notifications` every 30s and feeds into local panel
- [ ] DND hours: during DND, suppress sound + push, but still store in panel
- [ ] Settings saved to `ncc-settings` with keys:
  - `browserPush`, `notificationSound`, `notifyApps: { calendar: bool, todo: bool, agent: bool, system: bool }`, `dndStart`, `dndEnd`

## Files Expected to Touch
- `public/index.html` — add notification settings markup in settings view
- `public/js/app.js` — add settings wiring + agent notification polling
- `public/js/notifications.js` — respect DND + push toggle + sound toggle
- `public/css/base.css` — settings toggle/checkbox styles (reuse existing toggle class)

## Test
- Enable browser push → permission prompt → granted → toast "Push enabled"
- Trigger notification during DND → appears in panel, no sound, no push
- Agent POST to `/api/agent/notify` → frontend poll picks it up within 30s
