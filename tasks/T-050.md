# T-050 — Bookmark Manager

## Goal
A dedicated app for saving, organizing, and searching web bookmarks. Replaces scattered browser bookmarks with a dashboard-native solution.

## Acceptance Criteria
- [ ] Add bookmark: URL + title + optional description + tags + folder
- [ ] Auto-fetch: attempt to pull `<title>` and `<meta name="description">` from URL via proxy (or user pastes manually)
- [ ] Favicon: display site favicon (fetch from `https://www.google.com/s2/favicons?domain=...`)
- [ ] Folders: collapsible sidebar folder tree (Work, Personal, Tools, etc.)
- [ ] Tags: filter by tag, tag cloud view
- [ ] Search: full-text search across titles, descriptions, URLs
- [ ] Import: parse Netscape HTML bookmarks export format
- [ ] Export: generate Netscape HTML bookmarks file for browser import
- [ ] Dashboard widget: "Recently saved" or "Most visited" horizontal scroll
- [ ] Agent command: "bookmark this link", "find my docker bookmarks"

## UI Details
- App layout: sidebar folders + main grid/list of cards
- Card: favicon + title + truncated URL + tags + folder badge
- List view option: compact rows with sortable columns (date, title, folder)
- Add modal: URL input, auto-fill title/description fetch button, tag chips, folder dropdown
- Empty state: "Save your first bookmark 📎"

## Tech Notes
- Schema: `{id, url, title, description, folder, tags[], createdAt, visitCount, faviconUrl}`
- Storage: `localStorage` `ncc-bookmarks` (metadata only, not heavy)
- Favicon service: `https://www.google.com/s2/favicons?domain=example.com&sz=32`
- Import parser: simple regex on Netscape format `<DT><A HREF="...">title</A>`
- No external bookmark syncing (local only)

## Agent Note
Standalone app following the 7-file new-app wiring pattern. Start with CRUD + folders, add import/export and search in sub-tasks if needed. Good candidate for a SHORT task (~150 lines).
