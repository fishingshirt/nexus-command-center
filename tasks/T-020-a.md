# T-020-a — Introspection Prompt: Agent Reasoning Log

## Status
`PENDING`

## Goal
Before selecting which PENDING task to work on each wake cycle, the agent must output a brief **Introspection Log** that explains why it picked the task. This log becomes a persistent artifact (appended to `WHITEBOARD.md` Agent Notes and optionally stored server-side) and is also visible in the dashboard's Agent Stats panel.

## Rationale
> "make the AI think and explain before it acts, not just mindlessly check off tasks." — User directive

This forces the agent to evaluate task priority against the **PROJECT.md** product principles, not just `WHITEBOARD.md` row order. It creates accountability and lets the user audit agent decision-making.

## Acceptance Criteria
- [ ] Agent outputs an Introspection Block before coding on every cycle.
- [ ] Block contains at minimum: (a) top 3-5 candidate tasks considered, (b) reason for selection, (c) dependency/blocker analysis, (d) estimated effort, (e) alignment with phase gate (PROJECT.md §8).
- [ ] Block is appended to `WHITEBOARD.md` under a collapsible "Agent Introspection Log" section.
- [ ] Block is also POSTed to `/api/agent/introspection` and stored in `data/agent-introspection.jsonl`.
- [ ] Dashboard Agent Stats panel shows the **last 3 introspection entries** in a readable format.
- [ ] If the agent skips a HIGH priority task for a MEDIUM, it must justify the skip explicitly.
- [ ] Introspection is NOT sent to Telegram (too noisy). It stays in the dashboard and whiteboard.
- [ ] Format is markdown, 150-300 words max. No rambling.

## Introspection Template
```markdown
### Introspection — Cycle <timestamp>

**Candidates Considered:**
1. T-025 Smart PDF Editor (HIGH) — Blocked: requires file-upload server extension not yet built.
2. T-031 Cloud Vault (HIGH) — Blocked: depends on T-013 server-side persistence layer.
3. T-032 Email Agent (HIGH) — OAuth flow is complex; >200 lines, needs sub-tasks first.
4. T-050 Bookmark Manager (LOW) — Short, self-contained, pure client-side. High daily utility.

**Selection: T-050** — Lowest friction, zero blockers, fits "Phase 2 completion" priority. Building it advances the "Zero-Friction Entry" principle.

**Estimated Effort:** ~120 lines, 1 cycle.

**Phase Gate Alignment:** Phase 2 (Integration). T-050 is a utility that supports multiple apps (notes, todo, calendar all benefit from saved links).
```

## Server API
```
POST /api/agent/introspection
Body: { cycleId, timestamp, candidates[], selected, reason, phaseAlignment, estimatedLines }
Response: { ok: true, id }

GET /api/agent/introspection?limit=N
Response: { entries: [...] }
```

## Data Schema (per line in `data/agent-introspection.jsonl`)
```json
{
  "id": "<uuid>",
  "cycleId": "<cron-timestamp>",
  "timestamp": "2026-05-11T00:00:00Z",
  "candidates": [
    { "id": "T-XXX", "priority": "HIGH", "blocker": "string or null" }
  ],
  "selected": "T-050",
  "reason": "string",
  "phaseAlignment": "Phase 2",
  "estimatedLines": 120,
  "wordCount": 180
}
```

## UI Details
- Introspection log in Agent Stats: collapsible accordion, latest first.
- Use `JetBrains Mono` 12px for the body, `Inter` 13px for headers.
- Candidate list uses color-coded badges: `HIGH` = red, `MEDIUM` = amber, `LOW` = green.
- "Selected" task is bold with a `→` arrow prefix.
