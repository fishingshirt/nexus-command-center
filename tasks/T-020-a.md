# T-020-a: AI Task Suggester — JS App Module + Wiring

## Status
`PENDING`

## Goal
Build the frontend JS module for the AI Task Suggester app and wire it into the dashboard.

## Acceptance Criteria
1. Create `public/js/apps/ai-suggester.js`:
   - Export `initSuggester()` and `openSuggester()`
   - On open: render a view with:
     - Header: "AI Suggester" + refresh button + settings gear
     - Loading state: pulsing skeleton cards
     - Results grid: suggestion cards (title, description, priority badge, category tag)
     - Actions per card: "Add to To-Do" (creates todo item), "Dismiss" (stores ID in dismissed set)
     - Empty state: "No suggestions right now. Tap refresh to ask the AI."
   - Fetch from `/api/ai/suggest` with `POST { context: { recentEvents, pendingTodos, notesCount } }`
   - Cache response in `localStorage` (`ncc-ai-suggester-cache`) for 1 hour
   - Dismissed IDs stored in `localStorage` (`ncc-ai-suggester-dismissed`) as array
   - "Add to To-Do" writes to `localStorage` (`ncc-todo`) via the existing todo adapter (or direct localStorage if adapter unavailable)
2. Wire in `public/js/app.js`:
   - Add `import { initSuggester, openSuggester } from './apps/ai-suggester.js';`
   - Add `initSuggester();` call inside `initApp()`
   - Add `{ id: 'ai-suggester', name: 'AI Suggester', icon: '💡', path: 'ai-suggester' }` to `APP_REGISTRY`
3. Wire in `public/index.html`:
   - Add `<div id="view-ai-suggester" class="view" aria-label="AI Suggester">` inside `<main>`
   - Include suggester view markup (or generate via JS)
   - Add `<link rel="stylesheet" href="css/apps/ai-suggester.css">`

## Files Expected to Touch
- `public/js/apps/ai-suggester.js` (new)
- `public/js/app.js` (import + registry + init call)
- `public/index.html` (view shell + CSS link)

## Dependencies
None (uses existing `_ollama_generate` on server later; T-020-c adds the endpoint).
