# T-034-c-1 — Calendar Conflict Detection & Badge

## Goal
Detect when an event was modified on both the dashboard and Google Calendar since the last sync, and surface it to the user with an amber conflict badge in the calendar views.

## Acceptance Criteria
- [ ] Each local event stores `lastSyncedAt` and `lastModifiedAt` timestamps.
- [ ] Inbound sync (`gcal-sync.js` `mergeEvents()`) compares Google `updated` with local `lastModifiedAt`. If both sides changed since `lastSyncedAt`, mark the local event with `conflict: true`.
- [ ] Month, week, and day views render a small amber dot (`●`) on any event chip that has `conflict === true`.
- [ ] The calendar header shows a conflict counter (e.g. "3 conflicts") when any conflicted events are visible.
- [ ] Saving an event in `calendar.js` updates `lastModifiedAt = Date.now()`.
- [ ] `mergeEvents()` sets `lastSyncedAt = Date.now()` on all events it updates.

## Files to Touch
- `public/js/apps/calendar.js` — update `saveEvent()`, add conflict chip rendering, add conflict counter in header.
- `public/js/apps/gcal-sync.js` — update `mergeEvents()` to set timestamps and `conflict` flag.
- `public/css/apps/calendar.css` — add `.conflict-badge` amber dot styles.

## `localStorage` Keys / Schema
- Event objects gain fields:
  - `lastSyncedAt`: number (ms timestamp)
  - `lastModifiedAt`: number (ms timestamp)
  - `conflict`: boolean (only present when true)
