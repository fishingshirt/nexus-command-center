# T-043-d — Quick Capture: Agent Command Queue + Integration

## Goal
Add an agent command queue endpoint so external agents (cron, Telegram, voice) can inject quick-capture commands into the user's dashboard.

## Files To Touch
1. `scripts/nexus-server.py` — add POST `/api/agent/capture` endpoint (see references/agent-command-queue-pattern.md)
2. `public/js/apps/quick-capture.js` — add polling function and notification toast
3. `public/js/app.js` — optionally add server-side settings wiring if needed

## Acceptance Criteria
- [ ] Server endpoint `POST /api/agent/capture` accepts JSON: `{ mode, title?, body?, date?, time?, priority?, tags? }`
- [ ] Valid modes: `note`, `event`, `todo`, `voice`
- [ ] Server stores the command in a queue file `~/.hermes/nexus-capture-queue.json` (max 50 items, FIFO overflow)
- [ ] Server responds with `{ ok: true, id }` or `{ ok: false, error }`
- [ ] Frontend polls `GET /api/agent/capture/poll` every 30 seconds when dashboard is active
- [ ] Poll returns `{ ok: true, items: [...] }` where items are unacknowledged captures
- [ ] Frontend applies each item by calling the same save functions as manual capture (notes/event/todo)
- [ ] After processing, frontend acknowledges with `POST /api/agent/capture/ack` sending `{ ids: [...] }`
- [ ] When an agent-captured item is processed, show toast: `📥 Agent added: <title>`

## Queue File Schema
```json
{
  "queue": [
    {
      "id": "uuid",
      "mode": "note",
      "title": "...",
      "body": "...",
      "createdAt": "...",
      "acknowledged": false,
      "ackedAt": null
    }
  ]
}
```

## Security
- No authentication required (this is a local app), but sanitize all inputs before writing to JSON
- Limit field lengths: title 200 chars, body 5000 chars, tags array max 10 strings
- Escape any HTML in the toast (already done by toast function using textContent)

## Pitfall Avoidance
- Do NOT place the POST handler inside `do_GET()` in the server — it must be in `do_POST()`
- Do NOT use `default=str` on `json.dumps` unless handling datetime objects explicitly
- Ack endpoint must handle batch IDs, not one at a time, to save HTTP overhead
