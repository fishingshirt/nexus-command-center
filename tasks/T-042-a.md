# T-042-a: Notification API Core

## Status
`PENDING`

## Goal
Build the shared notification engine that all apps use to emit alerts. This is the foundation — no UI yet, just the API and persistence layer.

## Acceptance Criteria
- [ ] `public/js/notifications.js` exports:
  - `notify({ title, body, app, priority = 'normal' })` — adds to panel + optional browser push + optional sound
  - `getNotifications()` — returns array of notification objects
  - `markRead(id)`, `dismiss(id)`, `clearAll()`
- [ ] `localStorage` key `ncc-notifications` with max 100 items (FIFO eviction on overflow)
- [ ] Notification schema: `{ id, title, body, app, priority, read, timestamp }`
- [ ] Sound: short synthesized beep using Web Audio oscillator (0.2s, 880Hz) — only if user preference allows
- [ ] Browser push: call `new Notification(title, { body, icon })` only if permission granted and user toggle is on
- [ ] Backend endpoint `POST /api/notifications` adds a notification to the server-side queue (`~/.hermes/nexus-notifications.json`)
- [ ] Backend endpoint `GET /api/notifications` returns last 50 notifications
- [ ] Deduplication: same `(title, body, app)` within 10 seconds = single notification

## Files Expected to Touch
- `public/js/notifications.js` (new)
