# T-040-c: Dashboard Widget Grid — Widget Factories (Weather, Calendar, To-Do, Agent Stats)

## Goal
Implement real widget factory functions that pull live data from existing apps.

## Acceptance Criteria
1. In `public/js/widgets/grid.js` or dedicated `public/js/widgets/factories.js`:
   - `weatherWidget(config)` → card showing current temp + condition for default city
   - `calendarTodayWidget()` → card listing today's events from `localStorage` `ncc-calendar-events`
   - `todoCountWidget()` → card showing pending count + top 3 items
   - `agentStatsWidget()` → card showing last wake time, task count, server status
   - Each widget returns a DOM element with `data-widget-type` and auto-refresh logic
2. In each app's JS (`weather.js`, `calendar.js`, `todo.js`, `app.js` stats panel), expose a `getWidgetData()` helper the factory can call.
3. No duplicate data logic — widgets are thin wrappers around existing app stores.

## Files Expected to Touch
- `public/js/widgets/factories.js` (new)
- `public/js/apps/weather.js` (add `getWidgetData`)
- `public/js/apps/calendar.js` (add `getWidgetData`)
- `public/js/apps/todo.js` (add `getWidgetData`)
- `public/js/app.js` (expose stats for agent widget)
