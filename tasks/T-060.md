## Status
`PENDING`

## Severity

## Goal
Implement a two-stage quality assurance pipeline that runs on *every* code edit. Stage 1 (Quality Check) is fully automated ŌĆö syntax, cross-reference, route, and logic validation. Stage 2 (Quality Approval) is agent-reviewed ŌĆö logic sanity, functional test, site verification, then sign-off. Items that fail either stage are blocked from merge.

### Queues
All queues are stored **server-side** in `~/.hermes/nexus-quality-queue.json` with an append-only log so they survive git resets.

### Audit Log
Every check/approval writes to `~/.hermes/nexus-audit-log.ndjson` (one JSON object per line, never deleted).

## Stage 1 ŌĆö Quality Check (Automated)

### Trigger
Every commit or file change to `public/` or `scripts/` triggers Stage 1.

### Checks

#### 1.1 Syntax Validation
| Check | Tool | File |
|-------|------|------|
| JS syntax | `acorn.parse` (Node) or `node --check` | all `public/js/**/*.js` |
| CSS syntax | `postcss-safe-parser` or custom | all `public/css/**/*.css` |
| HTML well-formed | `html-validate` or `xmllint` | `public/index.html` |
| JSON schema | `JSON.parse` | all `tasks/*.md`, `manifest.json`, `WHITEBOARD.md` |

#### 1.2 Cross-Reference Validation
- Every `document.getElementById('X')` in JS must have a matching `<div id="X">` in HTML
- Every `document.querySelector('.X')` in JS must have a matching CSS rule `.X`
- Every `href="#app"` or `data-app="app"` must have a matching `#view-app` or register in app registry
- Every theme CSS must define the core variable set (`--bg`, `--text`, `--accent`, `--surface`, etc.)
- Every server endpoint mentioned in JS (`/api/...`) must exist in `nexus-server.py`
- Every task in `WHITEBOARD.md` must have a `tasks/T-XXX.md` file

#### 1.3 Module Import Validation
- Every `import { X } from './path.js'` must have an exported `X` in the target file
- No circular dependency loops in the module graph

#### 1.4 Server Route Validation
- Every `do_GET`/`do_POST` path in `nexus-server.py` is unique
- Every route has a response schema (returns `{'ok': bool}` or similar
- No stray `print()` in server handlers (use `json` responses only)

#### 1.5 Security Scan
- No hardcoded API keys, tokens, or secrets in any committed file
- No `innerHTML = text` with unescaped user inputs
- No `eval()` or `Function()` with dynamic user strings
- No `alert(document.cookie)` or similar XSS vectors

## Stage 2 ŌĆö Quality Approval (Agent Review)

### Trigger
Stage 2 ONLY runs when Stage 1 passes with `overall: "PASS"`.

### Checks

#### 2.1 Logic & Edge-Case Review
- [ ] Read every changed file line-by-line
- [ ] Verify state management uses correct `localStorage` keys
- [ ] Check for off-by-one, race conditions, null-reference errors
- [ ] Verify async/await error handling (`.catch()` or `try/catch`)
- [ ] Check for memory leaks (unremoved event listeners, uncleared intervals)
- [ ] Verify the change doesn't break existing features (cross-impact)

#### 2.2 Functional Test Plan
- [ ] Open `http://localhost:8080` in a browser
- [ ] Test the specific feature/bug area being changed
- [ ] Test at least 3 related edge cases
- [ ] Test on mobile viewport (Ōēż480px) AND desktop (Ōēź1024px)
- [ ] Test in the 2 most used themes (Jarvis + Professional)
- [ ] Test offline behavior (disable network, reload)

#### 2.3 Site Load Verification
- [ ] No console errors on initial load
- [ ] No FOUC (flash of unstyled content) ŌĆö check CSS blocking
- [ ] Service Worker registers without error
- [ ] All app cards visible on dashboard
- [ ] Navigation between views works

#### 2.4 Regression Check
- [ ] Run the existing `validate-js.js` Node checker
- [ ] Verify no new syntax errors across all JS files
- [ ] Run `curl` smoke tests on all `/api/*` endpoints
- [ ] Confirm `nexus-server.py` starts and serves without crash

## Queue System

### Server-Side Queue File
`~/.hermes/nexus-quality-queue.json`
```json
{
  "version": 1,
  "queue": [
    {
      "id": "QC-001",
      "status": "QC-PENDING",
      "commit": "abc1234",
      "files": ["public/js/app.js"],
      "created": "2026-05-10T12:00:00Z"
    },
    {
      "id": "QC-002",
      "status": "QC-PASSED",
      "commit": "def5678",
      "files": ["public/css/base.css"],
      "qa_id": "QA-002",
      "created": "2026-05-10T11:00:00Z"
    },
    {
      "id": "QA-002",
      "status": "QA-APPROVED",
      "commit": "def5678",
      "files": ["public/css/base.css"],
      "approved": "2026-05-10T13:00:00Z",
      "merged": "2026-05-10T13:05:00Z"
    }
  ]
}
```

### Lifecycle
| Status | Meaning |
|--------|---------|
| `QC-PENDING` | Agent just committed, waiting for auto-run |
| `QC-RUNNING` | Stage 1 in progress |
| `QC-PASSED` | Stage 1 passed, queued for Stage 2 |
| `QC-FAILED` | Stage 1 failed, needs fixes |
| `QA-PENDING` | Stage 2 waiting for agent |
| `QA-IN-REVIEW` | Agent currently reviewing |
| `QA-PASSED` | Stage 2 passed, ready to merge |
| `QA-FAILED` | Stage 2 found bugs, back to fix |
| `MERGED` | Committed to `main` and pushed |

## Backlog Audit (Legacy Files)

All existing source files (43 total) must be retroactively audited. These go into a **backlog queue** with no commit attached.

### Backlog Queue Format
```json
{
  "id": "QC-BACKLOG-001",
  "status": "BACKLOG-PENDING",
  "files": ["public/js/apps/weather.js", "public/css/apps/weather.css"],
  "reason": "NEVER_AUDITED",
  "priority": "HIGH",
  "created": "2026-05-10T12:00:00Z"
}
```

### Priority Rules
- Any file touching the Hermes Chat bridge (T-059): `CRITICAL`
- Any file touching auth/security: `CRITICAL`
- Any file in `public/js/apps/` with user CRUD: `HIGH`
- Any file in `public/css/themes/`: `MEDIUM`
- Any file in static assets (`assets/icons/`): `LOW`

## Automation Scripts

### `scripts/nexus-qc.py`
Runs all Stage 1 checks. Writes to `~/.hermes/nexus-quality-queue.json`.

### `scripts/nexus-qa-checklist.py`
Reads the queue, identifies what's ready for Stage 2, writes a `QA-XXX.md` summary for the agent.

### `nexus-server.py` Endpoints
- `GET /api/quality/queue` ŌĆö shows current queue status (for dashboard view)
- `GET /api/quality/result/:id` ŌĆö shows QC/QA result JSON
- `POST /api/quality/approve` ŌĆö agent submits sign-off
